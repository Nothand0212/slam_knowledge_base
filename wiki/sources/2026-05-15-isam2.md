# iSAM2: 基于 Bayes Tree 的增量平滑与建图

> Kaess 等人 IJRR 2012 发表，提出用 **Bayes Tree** 替代 Bayes Net 作为因子图推理的数据结构，
> 实现了高效的增量非线性优化。仅受新测量影响的因子才重线性化，是 GTSAM 库中 iSAM2 的核心理论基础。

## 论文信息

- **作者**：Michael Kaess, Hordur Johannsson, Richard Roberts, Viorela Ila, John J. Leonard,
  Frank Dellaert (MIT / Georgia Tech)
- **发表**：The International Journal of Robotics Research (IJRR), vol. 31, no. 2, pp. 216-235, 2012
- **DOI**：[10.1177/0278364911430419](https://doi.org/10.1177/0278364911430419)
- **代码**：iSAM2 集成于 [GTSAM](https://github.com/borglab/gtsam) 库（`ISAM2` 类）

## 核心贡献

1. **Bayes Tree 数据结构**：从因子图通过变量消除（Variable Elimination）得到 Bayes Net，再转换为
   Bayes Tree（clique tree）。树结构天然支持增量更新 —— 仅修改受新因子影响的分支。

2. **增量重线性化（Fluid Relinearization）**：并非每次新测量到来就全量重优化，而是只找出受影响的
   cliques（根据"top of Bayes Tree"向下追溯），对这些局部 cliques 重新做线性和消除。

3. **增量变量排序**：新变量插入时不破坏已有排序结构（使用 COLAMD 等启发式），保持 Bayes Tree 的
   平衡性和稀疏性。

4. **信息矩阵的增量更新**：新因子 = 顶部的局部线性化 + 底部受影响 cliques 的再消除，形成新的
   Bayes Tree，等价于从零批量优化但计算量大幅缩减。

5. **与 iSAM 的关系**：iSAM 是周期批量优化（每次更新信息矩阵的一列），iSAM2 是真正意义上仅处理
   "差分"的增量求解器，时间复杂度与更新大小相关而非全问题规模。

## 关键数学公式

**因子图因式分解（概率模型）**：

$$p(\mathcal{X}|\mathcal{Z}) \propto \prod_i f_i(\mathcal{X}_i)$$

其中 $f_i(\mathcal{X}_i)$ 为因子（约束函数），$\mathcal{X}_i$ 为因子涉及的状态变量子集。

**变量消除（Gaussian Elimination）**：

$$p(\mathcal{X}|\mathcal{Z}) \propto \prod_j p(x_j | S_j)$$

通过选择消除顺序，将联合分布转换为条件概率的乘积（Bayes Net），$S_j$ 为 $x_j$ 的分离子（separator）。

**Bayes Tree 构建**：

$$\text{Bayes Net} \xrightarrow{\text{identify cliques}} \text{Bayes Tree} = \{(C_k, S_k)\}$$

每个 clique $C_k$ 包含条件分布 $P(F_k|S_k)$，其中 $F_k$ 是前端（frontier，仅在此 clique 中出现的变量），
$S_k$ 是分离子（与父 clique 共享的变量）。

**iSAM2 增量更新**：

1. 新因子 $f'(\mathcal{X}')$ 加入
2. 找出 Bayes Tree 中与 $\mathcal{X}'$ 有交集的所有 cliques（"受影响的子图顶部"）
3. 移除受影响子图 → 与新因子合并 → 重排序 → 重消除 → 拼回 Bayes Tree

等价数学表达：新的联合分布为 $p'(\mathcal{X}) \propto p(\mathcal{X}) \cdot f'(\mathcal{X}')$，iSAM2
仅重新推导 $p'$ 中与 $f'$ 相关的条件部分。

## 架构概述

```
新测量 Z_new → 新因子 f_new(X_new)
                    ↓
        ┌─────────────────────────┐
        │    Bayes Tree (当前)      │
        │       Root               │
        │      /    \              │
        │   Clq_A  Clq_B ← X_new  │ ← 找出受影响 cliques
        │   /   \                  │
        │ Clq_C Clq_D              │
        └─────────────────────────┘
                    ↓
    移除受影响子图 → 局部重在线性 → 局部重消除
                    ↓
           新的 Bayes Tree（等价于从头批量优化）
                    ↓
         Δ^* = argmin Σ ||A'X - b'||² (局部求解)
                    ↓
              状态更新 (均值 + 协方差)
```

## 关键技术洞察

- Bayes Tree 的本质是"缓存高斯消除的中间结果"。在增量设置下，只有树中与新观测拓扑相交的部分需要
  重新计算，未受影响的分支完全复用。这正是 iSAM2 能在几十毫秒内处理数千个变量而批量方法需要数秒
  的原因。
- 实际工程中，重线性化触发的频率和程度（`ISAM2Params::relinearizeThreshold`）
  直接影响精度-速度 trade-off。GTSAM 默认 0.1 是经验最优值。
- iSAM2 是 Kimera-VIO、LIO-SAM 等系统的后端核心（`GTSAM::ISAM2`），直接驱动了视觉惯性 SLAM
  的实时非线性优化。
