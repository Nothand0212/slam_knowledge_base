# DM-VIO 论文精读

> DM-VIO 提出"延迟边缘化"策略，解决直接法 VIO 中尺度/重力方向/IMU bias 在边缘化时尚未收敛导致先验因子"锁定错误信息"的问题。在 EuRoC 和 TUM-VI 上取得 SOTA 精度。

## 核心贡献

### 1. 延迟边缘化（Delayed Marginalization）

传统 VIO 中，每帧新数据到达后立即边缘化最旧帧的信息（Schur Complement），形成先验因子。但如果此时 IMU 参数（尺度 scale、重力方向 gravity direction、IMU bias）尚未收敛，边缘化会在错误的线性化点处固定信息，产生的先验因子持续"拖拽"后续优化。

DM-VIO 的策略是：将待边缘化的帧放入延迟队列，等待 scale/gravity/bias 收敛后再执行边缘化。关键机制包括尺度变异系数监控和动态长度延迟窗口。

### 2. DSO + GTSAM 统一优化桥接

DM-VIO 的视觉前端继承 DSO（稀疏直接法，光度误差 BA），IMU 后端使用 GTSAM 因子图。两者的统一通过 Hessian 线性相加实现：

视觉部分（DSO）输出 Hessian $\mathbf{H}_\text{vis}$ 和梯度 $\mathbf{b}_\text{vis}$，IMU 部分（GTSAM）构造预积分因子 $\mathbf{H}_\text{imu}$ 和 $\mathbf{b}_\text{imu}$，统一线性系统为：

$$
(\mathbf{H}_\text{vis} + \mathbf{H}_\text{imu}) \Delta \mathbf{x} = \mathbf{b}_\text{vis} + \mathbf{b}_\text{imu}
$$

### 3. 动态视觉权重（Dynamic Visual Weight）

光度 RMSE 高时（运动模糊、大旋转、曝光变化），视觉因子可靠性下降，系统应更依赖 IMU。DM-VIO 引入自适应视觉权重：

$$
w_\text{vis} = \min\left(1, \frac{\tau^2}{\text{RMSE}_\text{photo}^2}\right)
$$

使用 $\tau$ 作为光度误差阈值，当 RMSE 超过 $\tau$ 时加权降低视觉因子贡献。这种设计使系统在挑战性运动中自动切换到 IMU 主导模式。

### 4. 两阶段初始化（Coarse-to-Fine Initialization）

DM-VIO 的初始化不依赖特定的静态/慢速运动模式，而是直接用所有观测进行优化：

1. **粗初始化**：短期纯视觉 BA + IMU 预积分，快速得到 scale/gravity 初始估计
2. **精细优化**：延迟边缘化窗口逐步扩展，scale/gravity 通过更多帧数据持续收敛

当 scale 的变异系数（$\text{std}/\text{mean}$）降至阈值以下时，标记为已收敛，后续允许执行边缘化。

### 5. 多延迟图架构（Multi-Delay Graph）

在 Marginalization 库中维护多个延迟图，每个对应不同的 delay 长度（如 3 帧/5 帧/8 帧）。系统根据当前参数收敛状态自动切换：
- 初始化阶段：使用长 delay，允许充分收敛
- 稳态运行：使用短 delay，减少窗口大小和计算量
- 参数波动（如 ba/g 变化大）：动态延长 delay

## 关键公式

**延迟边缘化可行性条件**：

当且仅当尺度 $s$、重力方向 $\mathbf{g}$、IMU bias $b_a, b_\omega$ 均充分收敛时，才执行对旧帧 $F_k$ 的边缘化：

$$
\frac{\sigma_s}{\bar{s}} < \epsilon_s \;\land\; \frac{\sigma_g}{\|\bar{\mathbf{g}}\|} < \epsilon_g \;\land\; \frac{\sigma_{b}}{\bar{b}} < \epsilon_b
$$

其中 $\sigma$ 和 $\bar{}$ 为滑动窗口内参数的均值和标准差，$\epsilon$ 为各参数的收敛阈值。

**视觉权重动态衰减**：

$$
H'_\text{vis} = w_\text{vis} \cdot H_\text{vis}, \quad w_\text{vis} = \min\left(1, \frac{\tau^2}{\text{RMSE}^2}\right)
$$

**边缘化先验因子的线性化点一致性（FEJ）**：

$$
\Phi_{\text{prior}}(\mathbf{x}) \approx \Phi(\mathbf{x}_{\text{FEJ}}) + \mathbf{J}_{\text{FEJ}} (\mathbf{x} - \mathbf{x}_{\text{FEJ}})
$$

DM-VIO 确保边缘化时使用首次估计 Jacobian（FEJ，First-Estimate Jacobian），避免不同线性化点导致的矛盾先验。

## 与知识库现有内容的联动

### 与 [[算法-DM-VIO]]、[[概念-延迟边缘化]] 的关系

现有实体页涵盖了延迟边缘化的动机和基本工作原理。本文档补充了论文层面的数学条件（收敛判断的变异系数）、多延迟图自动切换逻辑、动态视觉权重的完整推导，以及 DSO+GTSAM 统一优化的 Hessian 相加机制。

### 与 [[概念-Schur补与边缘化]] 的关系

传统边缘化（Schur Complement）是固定线性化点后消去变量的标准操作。DM-VIO 的"延迟边缘化"不是改变 Schur 本身的数学，而是**改变执行 Schur 的时机**（即变量何时被消去并固化信息），这是工程层面而非数学层面的创新。

### 与 [[方法-滑动窗口边缘化]] 的关系

传统滑动窗口边缘化（如 VINS-Fusion）在每帧到达后立即边缘化最旧帧，容易在初始化阶段引入错误先验。DM-VIO 的延迟策略推迟边缘化，直到参数收敛，从而避免传统方法的核心局限。

### 与 [[概念-视觉惯性初始化策略]]、[[方法-VIO初始化方法]] 的关系

DM-VIO 的两阶段初始化不需要静态/缓慢运动假设，在任意运动中均可初始化。这一特性和传统方法（VINS 的静止→视觉SFM→对齐）/（ORB-SLAM3 的大量 MAP 估计）形成鲜明对比。

### 与 [[算法-DSO]] 的关系

DM-VIO 的视觉前端继承自 DSO：稀疏直接法（2000 点）、光度误差 BA、滑动窗口优化、Immature Point 深度滤波。DM-VIO 在 DSO 基础上补充了 IMU 预积分因子和延迟边缘化，形成完整的单目 VIO。

### 与 [[概念-直接法光度误差]] 的关系

动态视觉权重的核心是判断光度误差的可靠性。DM-VIO 的 $w_\text{vis} = \min(1, \tau^2/\text{RMSE}^2)$ 是一种简单有效的自适应降权策略，比固定的权重比更鲁棒。

### 与 [[概念-因子图]]、[[组件-GTSAM]] 的关系

DM-VIO 的 IMU 后端使用 GTSAM 因子图：IMU 预积分因子、零初速因子、先验因子。延迟边缘化队列通过 DelayedGraph 类管理，最终通过 GTSAM 的 Marginalization 执行 Schur 补。

## Agent 实现提示

### 适用场景

延迟边缘化适用于需要在线的、对初始化鲁棒的 VIO/SLAM 系统，特别是直接法前端耦合 IMU 的场景。当后端使用因子图且需要滑动窗口边缘化时，延迟策略可以显著提升初始化阶段的精度和稳定性。

### 输入输出契约

- **输入**：单目图像序列（灰度）+ IMU 测量（加速度 $a_m$、角速度 $\omega_m$、采样频率 200+ Hz）
- **输出**：每帧相机位姿 $T_{WB} \in SE(3)$，速度 $v_{WB}$，IMU bias $b_a, b_\omega$，尺度 $s$，重力方向 $\mathbf{g}$
- **坐标**：World 坐标系（重力对齐）+ Body（IMU）坐标系
- **参数化**：位姿用 SE(3)，速度/偏置用 $\mathbb{R}^3$，尺度用 $\mathbb{R}^+$，重力用 $\mathbb{S}^2$

### 实现骨架（伪代码）

```pseudo
function dm_vio_pipeline(images, imu_measurements, K):
    # DSO 前端：稀疏直接法跟踪
    dso_tracker = DSOFrontend(K)
    # GTSAM 后端：IMU 因子图 + 延迟边缘化
    gtsam_backend = GTSAMBackend(imu_params)

    delayed_graph = DelayedGraph(max_delay=8)  # 边缘化队列
    scale_converged = False; gravity_converged = False

    for each frame (img, imus):
        # DSO 粗跟踪 + 点选择
        T_vis, points = dso_tracker.track(img)
        # IMU 预积分
        imu_factor = preintegrate(imus, T_vis, dt)

        # 构建统一因子图
        graph.add(imu_factor)
        graph.add(visual_factors(T_vis, points))

        # 动态视觉权重
        rmse_photo = compute_photometric_rmse(img, points)
        w_vis = min(1, tau^2 / rmse_photo^2)
        graph.scale_visual_weight(w_vis)

        # 检查收敛状态
        (s, g, bias) = graph.current_estimate()
        if var_coef(s) < eps_s and var_coef(g) < eps_g:
            # 参数收敛后：执行排队的边缘化
            delayed_graph.marginalize_converged()
        else:
            # 未收敛：推迟边缘化
            delayed_graph.enqueue_marginalize(oldest_frame)

    return trajectory, map, bias_estimate
```

### 实现注意事项

- **变异系数阈值 $\epsilon$** 的选择关键：过大则边缘化过早（信息不充分），过小则窗口过长（计算量大、帧间距大导致线性化点误差）。论文推荐 scale 阈值 ~0.05、gravity 阈值 ~0.03。
- **FEJ 必须严格遵守**：DM-VIO 在边缘化时强制使用节点的首次线性化点，否则会出现"来回拖拽"现象（新的优化结果与旧先验在同一变量的不同线性化点处冲突）。
- **DSO 的 Immature Point 和延迟边缘化的交互**：DSO 边缘化既涉及路标点的深度不确定性，也涉及关键帧位姿的不确定性；延迟边缘化需要配合 DSO 的 point marginalization 管理。
- **动态视觉权重的 $\tau$ 标定**：$\tau$ 应按照正常光度误差的分布选择（如 P95 值），避免对正常帧过度降权。
- **多延迟图切换的平滑性**：从长 delay 切换到短 delay 时需要保证旧图已被完全边缘化，否则会丢失约束连续性。
