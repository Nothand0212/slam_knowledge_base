# PX4 IMU 数据预处理管线

> PX4 作为最成熟的开源无人机飞控，其 IMU 预处理管线（硬件LPF + Butterworth二阶LPF + 静态/动态陷波 + 6面翻滚出厂标定 + 三阶多项式温度补偿 + 多IMU冗余投票）远比 SLAM 系统完善，可视为 IMU 信号预处理的上限参考。

## 基本信息

- **来源**：PX4 官方文档（filter_tuning / imu_factory_calibration / sensor_thermal_calibration）
- **类型**：技术文档综合
- **语言**：英文原文，中文整理

## 核心观点

1. **三级滤波架构**：硬件 LPF（芯片寄存器配置，抗混叠第一道防线）→ 软件 Butterworth 二阶 LPF → 静态/动态陷波滤波器，层层降噪
2. **陷波滤波是飞控核心能力**：针对多旋翼电机/旋翼振动，PX4 同时提供静态陷波和动态 FFT 自适应陷波；在已整理的主流 SLAM/VIO/LIO 工程中，这类输入预处理不常作为独立模块出现
3. **出厂标定远超 SLAM 水平**：6面翻滚测试 → scale factor + misalignment matrix + offset（`a_corrected = scale * (a_raw - offset)`，`w_corrected = misalignment * scale * (w_raw - offset)`）
4. **温度补偿**：三阶多项式 `TC_X0 + TC_X1*T + TC_X2*T^2 + TC_X3*T^3`，温控箱标定遍历 -10°C~+60°C；已整理的 SLAM/VIO/LIO 工程通常更依赖 bias 在线估计或传感器侧补偿
5. **振动隔离**：飞控与机架间用 3M 4011 等泡沫胶带减震，而不是尼龙柱硬连接——这是机械层面的第一道防线
6. **多 IMU 冗余与投票**：板载 + 外置 IMU 独立滤波后欧氏距离投票剔除离群值

## PX4 vs 主流 SLAM 系统对比

| 维度 | PX4 飞控 | 主流 SLAM 系统 |
|------|---------|--------------|
| 低通滤波 | Butterworth 二阶可配置 | 多数工程不作为独立数字滤波模块强调 |
| 陷波滤波 | 静态 + 动态 FFT 自适应 | 已整理工程中不常见 |
| 出厂标定 | 6 面翻滚 + 矩阵校正 | 多强调外参、时间偏移、bias 和噪声参数 |
| 温度补偿 | 三阶多项式 | 多依赖传感器侧补偿或 bias 在线估计 |
| 振动隔离 | 有硬件安装指导 | 通常由平台集成处理 |
| 多 IMU 冗余 | 投票机制 | 少数系统支持多 IMU，通常不以飞控投票方式实现 |

## 对 SLAM 的启示

PX4 的投入说明飞控对 IMU 信号质量要求极高（直接用于 PID 控制，延迟和噪声直接影响飞行稳定性）。SLAM 系统对 IMU 延迟容忍度更高（预积分在关键帧之间累积），但如果在高振动平台（无人机）上运行 SLAM，缺少陷波滤波器会导致陀螺仪数据被电机振动严重污染。

**最低推荐**：在无人机上做 SLAM 时，至少添加一个与电机转速匹配的陷波滤波器。

## 原文精彩摘录

- "硬件 LPF 是抗混叠的第一道防线，在 ADC 采样之前切除奈奎斯特频率以上的噪声"
- "动态陷波（Dynamic Notch）使用 FFT 实时检测主振动频率并自适应调节陷波位置"
- "PX4 文档强烈建议使用减震安装（如 3M 双面胶、硅胶减震柱），将飞控与机架的高频振动隔离"
