# Factor Graph

> 把变量和 measurement/constraint 分成两类节点的概率图模型；GTSAM 用它作为 SLAM、SfM 和 sensor fusion 优化问题的主表达。

## 在 GTSAM 中的意义

在 GTSAM 里，factor graph 通常不是直接手写矩阵，而是：

1. 用 `Values` 保存变量估计。
2. 用 `NonlinearFactorGraph` 保存 factors。
3. 每个 factor 根据当前 `Values` 计算 residual、Jacobian 和 noise-weighted error。
4. optimizer 或 `ISAM2` 线性化并求解增量。

## 工程直觉

- 变量节点：机器人位姿、速度、IMU bias、landmark、相机、lever arm 等。
- 因子节点：先验、里程计、IMU、GPS、重投影误差、stereo measurement、自定义传感器误差等。
- noise model 决定 measurement 对优化目标的权重。
- 初值质量直接影响非线性优化是否收敛到有用局部极小。

# GTSAM因子图

> GTSAM 中因子图（Factor Graph）的工程表达：将 SLAM 问题建模为变量节点和因子节点的二分图，通过非线性优化求解 MAP 估计。

## 核心组件

- **Values**：保存所有变量（位姿、速度、bias、landmark）的估计值
- **NonlinearFactorGraph**：容纳所有因子，每个因子计算 residual + Jacobian
- **NonlinearOptimizer**：LM/GN/Dogleg 批量优化器
- **ISAM2**：增量平滑与建图，支持在线逐帧更新

## 工作流程

1. 创建 `NonlinearFactorGraph` 和初始 `Values`
2. 添加因子：`PriorFactor`, `BetweenFactor`, `ImuFactor`, `GPSFactor` 等
3. 调用 `LevenbergMarquardtOptimizer(graph, values).optimize()` 或 ISAM2 增量更新
4. 从优化后的 `Values` 中提取结果
