# 直接法光度误差

> FAST-LIVO2 VIO 核心：不使用特征点，而是最小化稀疏图像块的光度残差 `r = τ_cur·I_cur(π(T_cw·p_w)) − τ_ref·I_warped_ref`。

## 残差含义

直接法假设同一个空间点在不同帧中的亮度经过曝光补偿后应保持一致。FAST-LIVO2 不先提取角点和描述子，而是在图像梯度较好的像素块上构造光度残差，并把这些残差直接写进 IESKF 更新。曝光参数 `tau` 用来吸收自动曝光或亮度变化带来的系统误差。

## 预处理边界

直接法里的图像预处理应优先理解为光度观测模型的一部分：

- DSO/DM-VIO 依赖 response、vignette 和 exposure，使像素强度更接近可比较的 irradiance。
- ROVIO/SVO 使用 patch 对齐和仿射亮度修正，缓解帧间曝光变化。
- FAST-LIVO2 把 inverse exposure 放进光度 residual 和 IESKF 状态。
- CLAHE/直方图均衡会改变局部非线性亮度映射，通常不应作为直接法默认预处理；若使用，更适合作为“破坏光度假设”的对照实验。

因此，直接法评估应看 photometric residual、patch NCC、曝光参数收敛和跟踪失败帧，而不是只看角点数量。

## 链式雅可比
- `∂r/∂δθ = J_img · ∂π/∂p_c · [Rci − Rci·skew(P_ic)]`
- `∂r/∂δp = J_img · ∂π/∂p_c · [Rci·R_wi^T]`
- `∂r/∂τ = I_cur(p)`（曝光雅可比）
- 粗到精金字塔优化，每级迭代至收敛后进入下级

## 优势
- 弱纹理场景优于特征点法
- 利用全图像梯度信息，不丢失细节
- 网格化每格保留一个最佳点控制计算量

## 失败模式

光度误差依赖亮度一致性，容易受运动模糊、滚动快门、强反光、动态物体、曝光突变和图像饱和影响。它也需要较好的初始位姿，否则投影点落到错误局部区域，金字塔优化也可能收敛到错误解。因此 FAST-LIVO2 中 LiDAR/IMU 先验对直接法视觉更新非常关键。

## Agent 实现提示

### 适用场景

当 agent 需要实现直接法视觉残差、patch alignment 或 LIVO 光度更新时，核心不是“提取特征点”，而是为参考 patch 和当前投影 patch 构造可解释的强度差，并把位姿、曝光/仿射亮度和鲁棒核的 Jacobian 一起接到优化或滤波器。

### 输入输出契约

- 输入：参考帧 patch、当前图像、参考点 3D 坐标/逆深度、当前位姿初值、相机模型、曝光/仿射亮度参数、图像梯度。
- 输出：光度残差向量、位姿 Jacobian、可选曝光/亮度 Jacobian、patch error、有效观测数。
- 坐标/单位：投影点在 processed pixel；图像强度可以是 raw intensity、response/vignette 校正后的 irradiance，或乘以曝光倒数后的归一化强度；三者不能混用。

### 实现骨架（伪代码）

```pseudo
function build_photometric_residual(ref_patch, curr_image, point_ref, state, camera, config):
    residuals = []
    H_pose = []
    H_exposure = []

    for pixel_offset in patch_pattern:
        p_cur = project(camera, transform(state.T_cur_ref, point_ref, pixel_offset))
        if p_cur outside image:
            continue

        I_cur, grad_cur = bilinear_sample_with_gradient(curr_image, p_cur)
        I_ref = ref_patch[pixel_offset]

        r = state.tau_cur * I_cur - ref_patch.tau_ref * I_ref
        J_pose = state.tau_cur * grad_cur * dproject_dpoint(p_cur) * dpoint_dpose(point_ref)
        J_tau = I_cur

        residuals.append(robust_weight(r, config.loss) * r)
        H_pose.append(J_pose)
        if config.estimate_exposure:
            H_exposure.append(J_tau)

    return residuals, H_pose, H_exposure, diagnostics
```

### 实现注意事项

- 光度残差前的“预处理”应保持单调、可解释或被显式建模；不要把 CLAHE 当作直接法默认增强。
- patch residual 的符号约定、曝光参数方向（exposure time 还是 inverse exposure）和 Jacobian 列顺序必须和状态定义一致。
- 图像金字塔每层都要同步梯度、尺度和投影阈值；低层用于捕获大位移，高层用于精化。
