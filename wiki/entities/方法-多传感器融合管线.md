# 多传感器融合管线 (fusions_slam)

> fusions_slam 的多模态传感器融合架构：基于 IESKF 的统一状态估计、纳秒级多传感器时间同步、IMU 传播-LiDAR 观测-RTK 位置/姿态/速度观测串行融合的完整 Pipeline 模式。

## 系统架构概览

├── wrapper/ros          # ROS 通信适配层 (数据格式转换 + 业务调度)
├── fusion_slam/modules  # 算法核心模块

核心设计原则：**数据通信与算法实现分离**——ROS 层负责 sensor_msgs 与内部数据结构的转换和时间同步，算法模块不依赖 ROS 接口。

## 纳秒级时间同步

`raw/codes/fusions_slam/src/fusion_slam/modules/front/front.cpp:L160-L224` — `syncMeasureGroupAdd`：

系统使用 `std::map<uint64_t, DataUnit>`（纳秒时间戳为 key）实现以 LiDAR 扫描为基准的多传感器对齐：

```
syncMeasureGroupAdd(msg):
    lidarBeginTime = clouds.front().timestamp (ns)
    lidarEndTime = lidarBeginTime + lastPoint.offset_time

    // 收集该 LiDAR 扫描时间窗口内的所有 IMU 数据
    while imus.front().timestamp in [lidarBeginTime, lidarEndTime]:
        msg.mapDatas[timestamp] = {type: IMU, data: imu}
        imus.pop_front()

    // 要求至少 5 个 IMU 测量
    if imu_count < 5: return false

    // 插入当前 LiDAR 扫描
    msg.mapDatas[lidarBeginTime] = {type: LIDAR, data: cloud}

    // 收集该时间窗口内的 RTK 数据
    while rtkPoses.front().timestamp in [lidarBeginTime, lidarEndTime]:
        msg.mapDatas[timestamp] = {type: RTK, data: rtkPose}
        rtkPoses.pop_front()
```

`DataUnit.type` 标识：0 = IMU，1 = LiDAR，2 = RTK。`mapDatas` 按时间戳自动排序，后续 propagate 按时间顺序遍历。

### 时间戳约定

- LiDAR 时间：首点时间戳（ROS header stamp）+ 逐点 `offset_time`（纳秒）
- IMU 时间：`header.stamp` 转 Unix 纳秒
- RTK 时间：GPS 周 + GPS TOW → Unix 时间戳（`gps2unix` 函数扣除闰秒 18s）

## IESKF 状态估计

### 状态向量 (18 维)

`raw/codes/fusions_slam/src/fusion_slam/modules/ieskf/ieskf.cpp:L10-L67`：

$$\mathbf{x} = [\underbrace{\delta\boldsymbol{\theta}}_{R^3}, \underbrace{\delta\mathbf{p}}_{R^3}, \underbrace{\delta\mathbf{v}}_{R^3}, \underbrace{\delta\mathbf{b}_g}_{R^3}, \underbrace{\delta\mathbf{b}_a}_{R^3}, \underbrace{\delta\mathbf{g}}_{R^3}]^T$$

状态定义：

| 分量 | 含义 | 初始协方差 |
|------|------|-----------|
| $\delta\boldsymbol{\theta}$ | 旋转误差状态 (so3) | $\text{diag}(10^{-4}, 10^{-4}, 10^{-4})$ |
| $\delta\mathbf{p}$ | 位置误差 | $\text{diag}(10^{-4}, 10^{-4}, 10^{-4})$ |
| $\delta\mathbf{v}$ | 速度误差 | $\text{diag}(10^{-3}, 10^{-3}, 10^{-3})$ |
| $\delta\mathbf{b}_g$ | 陀螺仪偏置 | $\text{diag}(10^{-5}, 10^{-5}, 10^{-5})$ |
| $\delta\mathbf{b}_a$ | 加速度计偏置 | $\text{diag}(10^{-5}, 10^{-5}, 10^{-5})$ |
| $\delta\mathbf{g}$ | 重力向量 | $\text{diag}(10^{-5}, 10^{-5}, 10^{-5})$ |

### IMU 传播 (Predict)

`raw/codes/fusions_slam/src/fusion_slam/modules/ieskf/ieskf.cpp:L70-L105`：

```pseudo
function predict(imu):
    dt = imu.t - X.t
    // 名义状态传播
    X.rotation = X.rotation ⊕ so3Exp((imu.gyro - X.bg) * dt)
    X.position += X.velocity * dt
    X.velocity += (R * (imu.acc - X.ba) + X.gravity) * dt

    // 协方差传播
    Fx = identity(18,18)
    Fx.block<3,3>(0,0) = so3Exp(-(imu.gyro - X.bg) * dt)    // δθ → δθ
    Fx.block<3,3>(0,9) = -J_right(-ω*dt) * dt               // δbg → δθ
    Fx.block<3,3>(3,6) = I * dt                               // δv → δp
    Fx.block<3,3>(6,0) = -R * skew(acc) * dt                 // δθ → δv
    Fx.block<3,3>(6,12) = -R * dt                             // δba → δv
    Fx.block<3,3>(6,15) = I * dt                              // δg → δv

    Fw.block<3,3>(0,0) = -J_right(-ω*dt) * dt               // ng → δθ
    Fw.block<3,3>(6,3) = -R * dt                              // na → δv
    Fw.block<3,3>(9,6) = I * dt                               // n_bg → δbg
    Fw.block<3,3>(12,9) = I * dt                              // n_ba → δba

    P = Fx * P * Fx^T + Fw * Q * Fw^T
```

### LiDAR 点-面观测更新

`raw/codes/fusions_slam/src/fusion_slam/modules/ieskf/ieskf.cpp:L107-L205` — `lidarObserve`：

对每个 LiDAR 点，在全局地图中查找最近 5 个点并拟合平面，构造 point-to-plane 残差：

$$r_i = \mathbf{n}^T \left( \mathbf{R} \cdot \mathbf{p}_i^{\text{imu}} + \mathbf{t} \right) + d$$

其中 $\mathbf{n}, d$ 是拟合平面参数，$\mathbf{p}_i^{\text{imu}}$ 是点在 IMU 坐标系下的坐标。

**IESKF 迭代更新**（最多 10 次迭代）：

```pseudo
for iter in 0..9:
    errorState = log(X_cur ⊕ X_prev^(-1))   // error state on manifold
    J_inv = I; J_inv.block(0,0) = J_right(δθ)
    curP = J_inv * P * J_inv^T

    // 计算残差 Z 和雅可比 H
    for each valid point:
        if planarCheck(nearPoints, pabcd, 0.1):
            pd = n^T * (R * p_i + t) + d
            s = 1 - 0.9 * |pd| / sqrt(||p_i||)  // 自适应权重
            if s > 0.9:
                Z(vi) = pd
                H(vi, 0:3) = -n^T * R * skew(p_i)  // rotation jacobian
                H(vi, 3:6) = n^T                     // translation jacobian

    K = (H^T * H + (P/λ)^(-1))^(-1) * H^T
    updateX = -K * Z - (I - K*H) * J_inv * errorState
    X_cur = X_cur ⊕ updateX
    if converge: break
```

自适应权重 $s = 1 - 0.9 \frac{|pd|}{\sqrt{|\mathbf{p}_i|}}$ 对远距离点和残差大的点降权，等效于软 Huber 损失。

### RTK 观测融合

`raw/codes/fusions_slam/src/fusion_slam/modules/ieskf/ieskf.cpp:L367-L411` — 三种观测类型：

**位置观测** (`positionObserve`, L382-L396)：
$$\mathbf{z}_p = \mathbf{p}_{\text{RTK}}, \quad \mathbf{H}_p = [\mathbf{0}_{3\times 3}, \mathbf{I}_{3\times 3}, \mathbf{0}_{3\times 12}]$$

**姿态观测** (`rotationObserve`, L367-L381)：
$$\mathbf{z}_r = \text{Log}(\mathbf{R}^{-1} \mathbf{R}_{\text{RTK}}), \quad \mathbf{H}_r = [\mathbf{I}_{3\times 3}, \mathbf{0}_{3\times 15}]$$

**速度观测** (`velocityObserve`, L397-L411)：
$$\mathbf{z}_v = \mathbf{v}_{\text{RTK}}, \quad \mathbf{H}_v = [\mathbf{0}_{3\times 6}, \mathbf{I}_{3\times 3}, \mathbf{0}_{3\times 9}]$$

RTK 更新在 propagate 中按时间顺序与 IMU 预测交替进行：

### 更新与重置 (Update-and-Reset)

`raw/codes/fusions_slam/src/fusion_slam/modules/ieskf/ieskf.cpp:L351-L365`：

```cpp
void IESKF::updateAndReset() {
    X.rotation = X.rotation ⊕ so3Exp(dX[0:3]);  // 重置名义状态
    X.position += dX[3:6];
    X.velocity += dX[6:9];
    X.bg += dX[9:12];
    X.ba += dX[12:15];
    X.gravity += dX[15:18];

    J.block<3,3>(0,0) = I - 0.5 * skew(dX[0:3]);  // 二阶梯近似
    P = J * P * J^T;  // 协方差修正
    dX.setZero();
}
```

## 工程边界与融合策略

| 策略 | 描述 |
|------|------|
| **紧耦合 vs 松耦合** | RTK 以观测形式直接进入 IESKF（紧耦合），而非先滤波再融合 |
| **时间同步** | 纳秒级 mapDatas 排序保证传感器数据按真实时间顺序处理 |
| **协方差一致性** | 统一 P 矩阵自动传播 IMU/LiDAR/RTK 之间的不确定性相关性 |
| **自适应权重** | LiDAR 残差权重 $s = 1 - 0.9\|pd\|/\sqrt{\|\mathbf{p}_i\|}$ 等效自适应 Huber 核 |
| **RTK 质量控制** | 外参链 `T_map_rtkPose = T_ENU_rtkPose * T_imu_ant^{-1}` 保证天线杆臂补偿 |

## Agent 实现提示

### 适用场景

当需要将 LiDAR、IMU、RTK/GNSS（可选相机）融合进统一的紧耦合状态估计器时，参考 fusions_slam 的 Pipeline 模式和 IESKF 实现。适用于自动驾驶建图定位、室外大场景 SLAM。不适合纯视觉系统或需要滑动窗口优化的场景（此时应选用 VINS-Mono 等方案）。

### 输入输出契约

- **输入**：
  - LiDAR 点云（Velodyne/Ouster），含逐点 `offset_time`
  - IMU 测量（角速度 + 加速度，硬件同步）
  - RTK 位姿（INSPVAX / NavSatFix / Odometry 三种格式可选）
  - 外参：`T_imu_lidar`(4×4), `T_imu_ant`(4×4, IMU→天线杆臂)
  - 配置 YAML：IESKF 协方差、LiDAR 线数、RTK 噪声、体素分辨率
- **输出**：
  - 18 维 IESKF 状态（位姿、速度、偏置、重力）
  - 协方差矩阵 P (18×18)
  - 当前帧点云（IMU 帧下）
  - 局部地图（IKD-Tree）
- **坐标系**：ENU 世界帧 ← IMU 帧 ← LiDAR 帧，时间戳为 Unix 纳秒

### 实现骨架（伪代码）

```pseudo
class MultiSensorFusionPipeline:
    function init(config):
        ieskf = IESKF(config)
        front = FrontBuffer()
        map = IKDTree()
        initStaticIMU(ieskf, duration=1.0)

    function run():
        while ros::ok():
            collectMessages()  // ROS spinOnce: LiDAR, IMU, RTK callbacks

            msg = syncMeasureGroupAdd(front)  // 以 LiDAR 扫描为基准对齐
            if msg is None: continue

            // IMU 传播 + RTK 观测交替
            for each sensor_data in msg.sortedByTime():
                if sensor_data.type == IMU:
                    ieskf.predict(sensor_data.imu)
                elif sensor_data.type == RTK:
                    ieskf.positionObserve(sensor_data.pose, noise)
                    ieskf.rotationObserve(sensor_data.pose, noise)

            // LiDAR 体素滤波
            cloud_filtered = voxelFilter(msg.cloud, leafSize=0.2)

            // IESKF LiDAR 观测更新
            ieskf.lidarObserve(cloud_filtered, map.kdtree, map.localMap)

            // 更新地图
            state = ieskf.getState()
            map.addPointsAndUpdate(cloud_filtered, state.R, state.t)

            publish(state, cloud_filtered)

    function syncMeasureGroupAdd(front):
        window = [clouds.front().timestamp, clouds.front().endTime]
        imus_in_window = extractIMUsIn(imus, window)
        if imus_in_window.count < 5: return None
        rtks_in_window = extractRTKsIn(rtkPoses, window)
        return sortedMsg(imus_in_window + rtks_in_window + [clouds.front()])
```

### 实现注意事项

1. **时间同步要求**：`readme:L76` 明确要求"所有数据均已进行时空同步（时间源统一，使用硬件同步时钟）"——IMU 和 LiDAR 必须使用同一时钟源，否则 propagate 中的时间戳对齐会失败。
2. **RTK 姿态质量**：当 `rtkType=0`（全模式）时，需要 RTK 提供准确的姿态（通常来自双天线 INS），若 RTK 只有位置信息，必须切换为 `rtkType=1`（仅位置），否则错误的姿态观测会严重破坏 IESKF 状态。
3. **LiDAR 逐点 offset_time**：`front.cpp:L167` 中 `lidarEndTime = lidarBeginTime + cloud.back().offset_time`——要求 LiDAR 驱动为每个点填充相对于首点的时间偏移，否则去畸变和同步失效。
4. **协方差重置**：IESKF 的 `updateAndReset` 使用 $J = I - 0.5 \cdot \text{skew}(\delta\theta)$ 的二阶梯近似进行协方差修正，这是 IESKF 与 ESKF 的关键区别——前者在误差状态重置后需要修正协方差的线性化点。
5. **平面拟合阈值**：`planarCheck(points, pabcd, 0.1)` 中 0.1 是平面拟合的残差阈值（单位：米），过大会接受非平面点（如边缘/角点），过小会拒绝所有远距离点（点云噪声大）。
6. **自适应权重门限 0.9**：`s = 1 - 0.9 * |pd| / sqrt(||p_i||) > 0.9` 等效于要求 `|pd| < 0.111 * ||p_i||`，即点面距离不超过点到传感器距离的 11%，这为近距离点和远距离点提供了不同的容忍度。
