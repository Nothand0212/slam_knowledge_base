# 隐式点-体素观测模型

> 隐式点-体素观测模型用局部体素的神经隐式特征预测扫描点残差和不确定度，目标是替代点到面/点到线等显式几何残差。

## 简介

传统 LiDAR scan-to-map 观测通常先为当前点找到局部邻域，再拟合平面、直线或高斯分布，最后构造点到几何 primitive 的残差。隐式点-体素观测模型把这个过程改写为：查询当前点所在体素及邻域体素的隐式特征，由网络直接预测“该点相对局部隐式几何应该如何修正”的残差向量和不确定度。

在 [[算法-P2V-SLAM]] 中，这个观测模型由两个网络配合实现：VE-Net 把体素内地图点编码为体素隐式特征，IR-Net 融合扫描点、邻域体素位置和体素特征，输出隐式残差与测量不确定度。后端仍使用 [[方法-IESKF滤波器]]，因此该模型需要提供可线性化、可加权的 measurement。

## 观测契约

- **输入**：当前 LiDAR 点、当前状态预测、LiDAR-IMU 外参、中心体素及邻域体素、各体素隐式特征。
- **输出**：隐式残差向量 `delta`、不确定度 `sigma`、用于 IESKF 的观测 Jacobian 或数值线性化结果。
- **坐标系**：点通常先从 LiDAR/body frame 经外参和状态变换到 world frame，再用 world frame 查询体素地图。
- **权重语义**：`sigma` 表示网络对局部几何预测的置信度，杂乱或训练分布外区域应产生更大的不确定度，降低观测权重。

## 与点到面残差的区别

点到面残差先显式估计法向量 `n` 和平面中心 `c`，再计算 `r = n^T(p_w - c)`。隐式点-体素观测不要求局部点集能稳定拟合平面，而是让网络从多个体素特征中学习局部几何约束。

这意味着模型表达力更强，但也把一部分几何可解释性和稳定性转移到了训练数据、网络结构和不确定度估计上。工程上不能只看平均轨迹误差，还要检查网络在退化场景、动态物体和训练分布外环境中的残差分布。

## Agent 实现提示

### 适用场景

当传统点到面/点到线残差在自然场景、植被、曲面、杂乱点云中观测不足，且系统可以接受神经网络推理开销时，可以考虑这个观测模型。若目标是极低算力嵌入式 LIO，或没有可代表部署环境的训练数据，应优先使用点到面、GICP 或 NDT。

### 输入输出契约

- 输入：deskew 后的 LiDAR 点 `p_l`、状态 `R_wb,t_wb`、外参 `R_bl,t_bl`、哈希体素地图、VE-Net 体素特征缓存、IR-Net 推理接口、邻域半径和观测数量上限。
- 输出：每个有效点的 `residual_delta`、`sigma`、`H_i`、`R_i^-1`、有效观测统计和网络推理耗时。
- 坐标/单位：点坐标单位为米；体素索引用 world frame；时间同步和 deskew 必须在观测构造前完成；`sigma` 应转换为测量协方差或信息权重。

### 实现骨架（伪代码）

```pseudo
function buildImplicitPointToVoxelMeasurements(scan_points, state, extrinsic, ivox_map, networks, config):
    measurements = []
    for p_l in sample_points(scan_points, config.max_observations):
        p_b = extrinsic.R_bl * p_l + extrinsic.t_bl
        p_w = state.R_wb * p_b + state.t_wb

        center_voxel = ivox_map.voxel_index(p_w)
        neighbor_voxels = ivox_map.neighbors(center_voxel, radius=config.neighbor_radius)
        voxel_features = []
        for voxel in neighbor_voxels:
            if ivox_map.feature_stale(voxel):
                ivox_map.feature[voxel] = networks.ve_net(ivox_map.points(voxel))
            voxel_features.append((voxel.relative_to(center_voxel), ivox_map.feature[voxel]))

        delta, sigma = networks.ir_net(point=p_w, features=voxel_features)
        if sigma > config.max_sigma:
            continue

        H_i = linearize_delta_wrt_error_state(delta, p_b, state, extrinsic)
        R_inv_i = inverse(clamp(sigma * sigma, config.min_var, config.max_var))
        measurements.append({residual: delta, H: H_i, R_inv: R_inv_i})

    return measurements
```

```pseudo
function ieskfUpdateWithImplicitResidual(state, covariance, measurements):
    for iter in 1..max_iter:
        H, r, R_inv = stack(measurements)
        dx = solve((H.T * R_inv * H + inv(covariance)), H.T * R_inv * r)
        state = inject_error(state, dx)
        if norm(dx.pose) < convergence_threshold:
            break
    return state
```

- iVoxMap 更新：点写入体素、固定点数、特征刷新阈值。

### 实现注意事项

- 不要把 IR-Net 输出当作普通 ICP 最近邻残差；它的 `sigma` 是观测可信度的一部分，必须进入权重。
- 采样策略会直接影响实时性和可观测性，不能只按随机点采样；应保留空间分布和高信息区域。
- 体素特征刷新频率要受控，特征过旧会拖累配准，过频会让 VE-Net 成为瓶颈。
- 训练集分布决定泛化边界；部署到矿区、森林、室内工厂前应单独做残差统计和定位漂移评估。
- 和点到面 IESKF 混用时，要明确 measurement block 的噪声单位、残差维度和 Jacobian 扰动约定。
