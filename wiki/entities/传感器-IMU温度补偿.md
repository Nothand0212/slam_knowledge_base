# IMU温度补偿

> PX4 使用三阶多项式拟合 IMU 偏置随温度的漂移曲线，通过温控箱标定获得系数，运行时根据芯片内部温度传感器实时补偿。

## 补偿模型

三阶多项式：
```
correction = TC_X0 + TC_X1*T + TC_X2*T^2 + TC_X3*T^3
```

其中 T 为 IMU 芯片内部温度传感器读数。

PX4 参数：
- `TC_A*` 系列：加速度计温度补偿系数（三阶）
- `TC_G*` 系列：陀螺仪温度补偿系数（三阶）

## 标定流程

1. 将飞控放入温控箱
2. 遍历工作温度范围（-10°C 至 +60°C）
3. 在每个温度点恒温后记录 IMU 偏置漂移
4. 离线拟合三阶多项式系数
5. 系数写入飞控参数

## 与 SLAM 的对比

在已整理的主流 VIO/LIO/SLAM 工程中，IMU 温度补偿通常不是独立的输入预处理模块；更多系统把陀螺仪和加速度计 bias 作为估计器状态在线估计，或者假设传感器驱动/硬件已完成基础补偿。

这不等于 SLAM 不需要温度补偿。对户外无人机、车载长时间运行或温差较大的平台，温度引起的 bias 漂移会改变预积分和状态传播质量。PX4 的三阶多项式方案可作为嵌入式 SLAM 的参考，但移植时需要同时验证：

- 温控箱标定数据是否覆盖实际工作温度范围。
- 补偿后是否仍需估计 residual bias。
- 温度补偿引入的参数更新是否影响 IMU 时间同步和滤波延迟。
