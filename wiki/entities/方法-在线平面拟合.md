# 在线平面拟合

> LiDAR-IMU 前端中直接在局部邻域拟合平面，用平面性检验生成 point-to-plane 约束，避免预先做固定 edge/planar 特征分类。

## 定义

在线平面拟合是在点云配准前端中，把当前点的近邻实时拟合为局部平面，并用点到平面的距离作为残差或特征有效性判断。它常见于 IESKF LIO 和轻量级 LiDAR odometry：相比 LOAM 式曲率分类，它更依赖局部地图几何和最近邻质量。

## 通用流程

1. 在 IVox3d 地图中查找 `NUM_MATCH_POINTS=5` 个最近邻点
2. 用约束 `n^T·p=-1` 的 `A.colPivHouseholderQr().solve(b)` 求解法向量
3. 输出归一化平面系数 `(a,b,c,d)` 满足 `ax+by+cz+d=0`
4. 验证所有邻点到平面距离 ≤ `ESTI_PLANE_THRESHOLD=0.1`
5. 距离有效性验证：`d_body > 9 * |plane_residual|`

## 工程案例

| 工程 | 实现特点 | 风险 |
|------|----------|------|
| Lightning-LM | 在 IVox3d 最近邻中实时拟合平面，不区分 edge/planar，用平面性验证替代固定特征分类 | 阈值和近邻质量直接影响残差稳定性 |
| fusions_slam `planarCheck` | 对 5 个邻域点做 QR 最小二乘，要求所有点到平面距离小于阈值，作为 IESKF point-to-plane 观测前的几何验证 | 原函数缺少显式退化检测，QR 奇异或近邻共线时需要额外保护 |

## 优势

- 不需要预先设定 edge/planar 分类阈值
- 自适应场景几何，特征质量由数据驱动
- 比传统 LOAM 曲率方法更少参数

## 设计注意

- 近邻数量过少时平面约束不稳定，过多时会跨越真实几何边界。
- 只检查点到平面距离不够，还应关注邻域协方差特征值或法向条件数。
- 平面残差进入 IESKF/优化器前，应和测量噪声、鲁棒核、退化检测一起设计。

## Agent 实现提示

### 适用场景

LiDAR-IMU 紧耦合前端中需要为每个点实时计算局部平面法向量：IESKF point-to-plane 观测构建、平面性验证替代固定 edge/planar 分类、体素地图内的局部几何拟合。适合对最近邻点集直接 QR 求解法向量的在线场景。

### 输入输出契约

- 输入：近邻点集 `point`（`PointVector`，至少 `MIN_NUM_MATCH_POINTS` 个），平面距离阈值 `threshold`（默认 0.1m）。
- 输出：四维平面向量 `pca_result = (nx, ny, nz, d)` 满足 `n·p + d = 0`，返回 `true` 表示通过所有邻点到平面距离检验。
- 前置条件：近邻 `point` 已在 IVox3d 体素地图中查询得到（典型 `NUM_MATCH_POINTS = 5`）；点坐标在同一坐标系（通常是世界系或 LiDAR 系）。
- 副作用：无，纯函数。

### 实现骨架（伪代码）

```text
function esti_plane(point, threshold):
    if point.size() < MIN_NUM_MATCH_POINTS: return false

    // 构架 n×3 矩阵 A 和 n×1 向量 b，求解 n·p = -1
    A = zeros(point.size(), 3)
    b = ones(point.size()) * (-1.0)
    for j in range(point.size()):
        A.row(j) = (point[j].x, point[j].y, point[j].z)

    // QR 分解求最小二乘解（n = normvec）
    normvec = A.colPivHouseholderQr().solve(b)

    // 归一化得到 (a, b, c, d)
    nn = normvec.norm()
    pca_result = (normvec(0)/nn, normvec(1)/nn, normvec(2)/nn, 1.0/nn)

    // 验证：所有邻点到平面距离是否低于阈值
    for each p in point:
        vec4 = (p.x, p.y, p.z, 1.0)
        if abs(pca_result.dot(vec4)) > threshold:
            return false

    return true
```

### 关键源码片段

`raw/codes/lightning-lm/src/core/lightning_math.hpp:L358-L414`

```cpp
template <typename T>
inline bool esti_plane(Eigen::Matrix<T, 4, 1>& pca_result, const PointVector& point,
                        const T& threshold = 0.1f) {
    if (point.size() < fasterlio::MIN_NUM_MATCH_POINTS) {
        return false;
    }
    Eigen::Matrix<T, 3, 1> normvec;
    if (point.size() == fasterlio::NUM_MATCH_POINTS) {
        Eigen::Matrix<T, fasterlio::NUM_MATCH_POINTS, 3> A;
        Eigen::Matrix<T, fasterlio::NUM_MATCH_POINTS, 1> b;
        A.setZero(); b.setOnes(); b *= -1.0f;
        for (int j = 0; j < fasterlio::NUM_MATCH_POINTS; j++) {
            A(j, 0) = point[j].x; A(j, 1) = point[j].y; A(j, 2) = point[j].z;
        }
        normvec = A.colPivHouseholderQr().solve(b);
        // ... (>5点时扩展为动态矩阵版本)
    }
    T n = normvec.norm();
    pca_result(0) = normvec(0) / n; pca_result(1) = normvec(1) / n;
    pca_result(2) = normvec(2) / n; pca_result(3) = 1.0 / n;  // d = 1/|n|
    for (const auto& p : point) {
        Eigen::Matrix<T, 4, 1> temp = p.getVector4fMap(); temp[3] = 1.0;
        if (fabs(pca_result.dot(temp)) > threshold) {
            return false;
        }
    }
    return true;
}
```

### 实现注意事项

- 求解的数学模型是 `n·p = -1`，归一化后 `d = 1/|n|`，满足 `nx*x + ny*y + nz*z + d = 0` 的平面标准形式。这与 `n·p = 0`（过原点平面）不同，需要仔细对齐下游 `point-to-plane` 残差的定义。
- 5 个近邻点恰为确定秩-3 平面所需的最小点数，当过拟合时需要至少 6 点以获得最小二乘平滑；当前代码在 `point.size() == NUM_MATCH_POINTS` 路径下不做最小二乘冗余检验。
- 仅检查点到平面距离（阈值 0.1m）是必要但不充分条件；近邻共线时 QR 近乎奇异，法向量不可靠。建议额外检查邻域协方差最小特征值或条件数。
- `colPivHouseholderQr` 有列主元选择，比普通 Householder QR 在近奇异情况下更稳健，但不能替代显式退化检测。
- 在 IESKF 迭代中，平面拟合在每个点的每次迭代中重复执行；可考虑缓存最近邻结果，在位姿增量小于体素尺寸时复用。
- 平面残差进入 IESKF 观测方程前，应配合鲁棒核（如 Huber）防止错误平面的杠杆效应。
