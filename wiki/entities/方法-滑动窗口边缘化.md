# 滑动窗口边缘化

> VINS-Fusion 的滑动窗口边缘化实现：KF 触发 → 构建信息矩阵 → Schur 补 → 线性化先验因子 → 接入 Ceres 优化。是滑窗优化器中最关键的工程机制。

## 整体流程

```
新帧到达
  │
  ├─ addFeatureCheckParallax() → 判断是否为 KF
  │
  ├─ KF=true  → marginalization_flag = MARGIN_OLD
  │   └─ 边缘化最老帧（保留其信息作为先验）
  │
  ├─ KF=false → marginalization_flag = MARGIN_SECOND_NEW
  │   └─ 边缘化次新帧（丢弃其视觉观测，保留 IMU 约束）
  │
  ├─ optimization()  → Ceres DENSE_SCHUR 求解
  │
  └─ slideWindow()   → 物理移除帧，更新数据结构
```

**源码锚点**: `raw/codes/VINS-Fusion/vins_estimator/src/estimator/estimator.cpp:L411-L427`, `L1122-L1200`

## 边缘化的两种模式

### MARGIN_OLD（KF 帧 → 删最老帧）

当当前帧被判定为 KF 时，滑窗已满，需要移除最老帧。最老帧连接了滑窗内外的信息，直接丢弃会损失约束。

**处理策略**：
1. 将与最老帧（Pose[0], SpeedBias[0]）相关的所有残差块收集到 `MarginalizationInfo`
2. 包括：IMU 预积分因子（帧 0→1）、视觉重投影因子、上一轮边缘化先验因子
3. 调用 `marginalize()` 执行 Schur 补
4. 把生成的 `MarginalizationFactor` 作为先验项加入下一轮优化

**关键源码**（`estimator.cpp:L1140-L1196`）:
```cpp
MarginalizationInfo *marginalization_info = new MarginalizationInfo();

// 传播上一轮先验：如果上一轮先验中涉及被边缘化的变量，将其纳入
if (last_marginalization_info && last_marginalization_info->valid) {
    // 找到上一轮先验中关联 Pose[0] 或 SpeedBias[0] 的参数块
    // 标记为 drop_set，将整个先验残差块加入本轮边缘化
}

// 加入 IMU 因子（帧0→1）
IMUFactor* imu_factor = new IMUFactor(pre_integrations[1]);
// drop_set = {0, 1}（标记 Pose[0] 和 SpeedBias[0] 为被边缘化）

// 加入视觉因子（所有包含帧0观测的特征）
for each feature with observation at frame 0:
    ProjectionFactor *f = ...
    // drop_set 取决于边缘化哪些变量

marginalization_info->preMarginalize();  // 线性化所有残差
marginalization_info->marginalize();     // Schur 补
```

### MARGIN_SECOND_NEW（非 KF → 删次新帧）

非 KF 时，次新帧的视觉观测信息量有限，但需要保留其 IMU 预积分约束。

**处理策略**：
1. 直接丢弃次新帧（Pose[WINDOW_SIZE-1]）的视觉观测
2. 将次新帧的 IMU 测量**合并**到前一帧的预积分中（`slideWindowNew`）
3. 不执行 Schur 补边缘化，避免引入不必要的先验约束

**关键源码**（`estimator.cpp:L1386-L1421`）:
```cpp
// 非 MARGIN_OLD：MARGIN_SECOND_NEW
if (frame_count == WINDOW_SIZE) {
    // 将 frame[WINDOW_SIZE] 的 IMU 数据合并到 frame[WINDOW_SIZE-1]
    for each IMU measurement at frame[WINDOW_SIZE]:
        pre_integrations[frame_count - 1]->push_back(dt, acc, gyr)
    // 状态传播
    Ps[frame_count - 1] = Ps[frame_count]
    Vs[frame_count - 1] = Vs[frame_count]
    ...
    slideWindowNew();  // 物理删除最老的特征帧
}
```

## Schur 补边缘化实现

**源码锚点**: `raw/codes/VINS-Fusion/vins_estimator/src/factor/marginalization_factor.cpp:L183-L311`

### 线性系统构建

将所有待边缘化残差收集后，构建大信息矩阵 A 和向量 b：

```
A = [A_mm  A_mr]    b = [b_m]
    [A_rm  A_rr]        [b_r]
```

其中 `m` 是被边缘化变量的维度，`r` 是保留变量的维度。

**多线程加速**（`ThreadsConstructA`）：
```cpp
// 4 线程并行构建 A 和 b
for each residual block:
    J_i = jacobian for parameter block i
    A.block(idx_i, idx_j) += J_i^T * J_j    // 对称更新
    b.segment(idx_i)     += J_i^T * residual
```

注意：SE(3) 位姿切空间是 6 维，但参数块是 7 维（x,y,z,qx,qy,qz,qw），需要 `localSize(7) = 6` 映射为切空间维度后再构建 H。

### Schur 补

```cpp
Eigen::MatrixXd Amm = 0.5 * (A.block(0,0,m,m) + A.block(0,0,m,m).transpose());
Eigen::SelfAdjointEigenSolver<Eigen::MatrixXd> saes(Amm);
// 正则化逆：特征值 < eps 的分量置零（处理秩亏）
Eigen::MatrixXd Amm_inv = saes.eigenvectors()
    * VectorXd((eigenvalues > eps).select(1/eigenvalues, 0)).asDiagonal()
    * saes.eigenvectors().transpose();

// Schur 补公式：A' = A_rr - A_rm * A_mm^{-1} * A_mr
A = Arr - Arm * Amm_inv * Amr;
b = brr - Arm * Amm_inv * bmm;
```

### 先验因子线性化

边缘化后产生的 `MarginalizationFactor` 作为 Ceres `CostFunction`：

```
residual = linearized_residuals + linearized_jacobians * dx
```

其中 `dx = current_params - linearization_point`，即相对于线性化点的增量。对 SE(3) 位姿，`dx` 使用李代数差：
```
dx = [p - p0, 2 * Log(q0^{-1} * q)]  // 6 维切空间
```

**关键约束**：先验因子是**一次线性化固定**（first-estimate Jacobian），后续优化不能重新线性化。这保证了边缘化的一致性。

## 数据结构关系

```
MarginalizationInfo
├── factors: vector<ResidualBlockInfo*>  // 所有参与边缘化的残差
│   ├── cost_function: ceres::CostFunction*
│   ├── parameter_blocks: vector<double*>  // 参数块地址
│   ├── drop_set: vector<int>  // 被边缘化的参数块索引
│   └── jacobians, residuals  // Evaluate() 后填充
│
├── parameter_block_size: map<addr → global_size>
├── parameter_block_idx:  map<addr → index>
└── linearized_jacobians, linearized_residuals  // Schur 补后
    │
    └─→ MarginalizationFactor (注入下一轮 Ceres 优化)
```

## 与 ORB-SLAM3 和 MSCKF 的对比

| 维度 | VINS-Fusion | ORB-SLAM3 | open_vins MSCKF |
|------|------------|-----------|-----------------|
| 边缘化方式 | Schur 补信息矩阵 | 关键帧剔除 | 零空间投影消去特征 |
| 被边缘化量 | 最老帧的位姿/速度/偏置 | 冗余 KF 的地图点 | 特征坐标（消去 3D 点） |
| 先验形式 | 线性化先验因子接入 Ceres | KF 删除后 BA 重优化 | 零空间投影后 EKF 更新 |
| 一致性保证 | FEJ（一次线性化固定） | 删除后重新 BA | MSCKF 可观测性约束 |
| 适用场景 | 滑窗优化 | 全局/局部 BA | EKF 滤波 |

## Agent 实现提示

### 适用场景
需要为基于 Ceres/g2o/GTSAM 的滑窗优化器实现边缘化模块。

### 输入输出契约
- **输入**：待边缘化的残差块列表（`ResidualBlockInfo`），每个包含参数块地址、Jacobian、残差、drop_set
- **输出**：线性化先验因子 `MarginalizationFactor`，可直接加入下轮优化的 Ceres::Problem

### 实现骨架（伪代码）

```
class MarginalizationInfo:
    function marginalize():
        // 1. 建立参数块索引（被边缘化变量在前，保留变量在后）
        m = total_dim_of_dropped_blocks
        n = total_dim_of_kept_blocks

        // 2. 对所有残差块 Evaluate，构建 H * dx = b
        A = zero_matrix(m+n, m+n);  b = zero_vector(m+n)
        for each residual_block:
            J_i = jacobian_of_block_i
            A += J_i^T * J_i
            b += J_i^T * residual_i

        // 3. Schur 补
        Amm_inv = pseudo_inverse(A[0:m, 0:m])  // 正则化处理秩亏
        A_reduced = A[m:, m:] - A[m:, 0:m] * Amm_inv * A[0:m, m:]
        b_reduced = b[m:] - A[m:, 0:m] * Amm_inv * b[0:m]

        // 4. 分解为先验 Jacobian 和残差
        eigendecompose(A_reduced) → V * S * V^T
        linearized_jacobians = sqrt(S) * V^T
        linearized_residuals = 1/sqrt(S) * V^T * b_reduced

    function priorResidual(current_params, linearization_point):
        dx = current_params ⊖ linearization_point  // 流形差
        return linearized_residuals + linearized_jacobians * dx
```

### 实现注意事项
- SE(3) 位姿用 6 维李代数参数化，**不是** 7 维四元数，需要使用 `localSize()` 映射
- Schur 补前对 Amm 正则化逆，处理秩亏（信息矩阵可能半正定）
- 线性化点必须与边缘化时一致（first-estimate Jacobian），后续优化中不能重线性化先验因子
- 多线程构建 H 矩阵时注意对称性（`A.block(i,j)` 更新后同步更新 `A.block(j,i)`）
