# 可微渲染

> 将渲染过程变成可反向传播的优化模块，使相机位姿和地图参数能通过图像/深度残差联合更新。

## 核心思想

可微渲染把“地图到图像”的过程写成可求导函数。给定相机位姿和地图参数，系统先渲染 RGB、深度或 silhouette，再和真实观测计算损失，并把梯度反向传播到位姿、SDF、神经特征或高斯参数。它把传统 SLAM 的“找对应点再最小化重投影误差”改成“直接最小化渲染误差”。

典型流程是：射线采样或 splatting → 查询地图表示 → 合成像素颜色/深度 → 计算光度与几何损失 → 对相机位姿和地图参数求梯度。Track 阶段通常固定地图只优化当前位姿；Map 阶段则固定或弱优化位姿，同时更新场景参数。

## 两种常见实现

NICE-SLAM 采用神经隐式表示和体积渲染。每条相机射线在空间中采样，MLP/SDF 解码器预测几何和颜色，再通过 alpha 合成得到像素监督。MonoGS 采用 3D Gaussian Splatting，把一组各向异性高斯投影到图像平面并做可微栅格化，渲染速度更高，更适合在线跟踪。

## 与 SLAM 的关系

可微渲染提供的是一种残差构造方式，而不是完整 SLAM 系统。它仍需要初始化、关键帧选择、地图增长、外点处理和位姿先验。对单目系统，尺度和深度歧义仍然存在；对 RGB-D 系统，深度监督能显著降低歧义但依赖深度质量。

## 局限

- 对曝光变化、运动模糊和动态物体敏感，需要鲁棒损失或 mask。
- 在线优化成本高，常需要 GPU 和关键帧采样策略。
- 梯度可能落入局部极小值，因此仍需要里程计、IMU 或粗位姿初始化。
- 渲染质量好不等于几何度量一致，机器人导航还要关注尺度、可通行空间和不确定性。

## Agent 实现提示

### 适用场景

当 Agent 需要把“地图参数/位姿优化”写成端到端可反传流程时使用可微渲染：输入相机和地图，渲染深度/RGB，再用观测误差反向更新。NICE-SLAM 代表射线采样和体积合成路线，MonoGS 代表 Gaussian splatting 路线；两者都要求良好的初始化和鲁棒 mask。

### 输入输出契约

- 射线输入 `rays_o/rays_d`: `Tensor[N_rays, 3]`，由相机内参、姿态和采样像素生成。
- 地图输入 `c, decoders` 或 GaussianModel：前者是特征网格和解码器，后者是可微高斯参数。
- 监督输入 `gt_depth/gt_color`: `Tensor[N_rays]` 和 `Tensor[N_rays, 3]`。
- 输出 `depth/color/uncertainty` 或 `render/depth/opacity`，loss 必须是标量并能反传到位姿或地图参数。

function differentiable_render_update(camera, map, observations):
    rays_o, rays_d, gt_depth, gt_color = sample_pixels(camera, observations)
    if map is implicit_grid:
        points = sample_along_rays(rays_o, rays_d, gt_depth)
        raw = decoder(points, feature_grid)
        pred_depth, uncertainty, pred_color = volume_composite(raw)
    else if map is gaussian_splat:
        pred = rasterize_gaussians(camera, map)
        pred_depth, pred_color = pred.depth, pred.render
    loss = depth_loss(pred_depth, gt_depth, valid_mask)
    loss += color_weight * color_loss(pred_color, gt_color)
    loss.backward()
    optimizer.step()

### 实现注意事项

- 渲染函数应返回可微张量；不要在 loss 前调用 `.detach()`、`.cpu().numpy()` 或非可微后处理。
- 深度为 0 的像素需要 mask，否则会把无效观测当成几何约束。
- Tracking 和 Mapping 的优化变量不同：前者多优化相机 pose，后者多优化地图特征/解码器/高斯参数。
- 可微渲染只定义残差，不自动解决动态物体、外点、尺度漂移和关键帧管理。
