# 点到点ICP

> 基于点到点欧氏距离残差（3D）的 ICP 变体，区别于点到平面（1D）ICP。

## 核心特点

每个 source 点到 target 最近点构成 3 维残差 `r = s - t`，Jacobian 为 3×6 矩阵 `J = [I | -hat(s)]`（left-perturbation on SE(3)）。每对对应关系贡献 3 个方程，信息量是点到平面的 3 倍，但缺少结构先验。

## 关键权衡

- **优点**：各向同性，不依赖法向量估计质量，退化鲁棒性更好
- **缺点**：收敛更慢（KISS-ICP 默认 500 次迭代 vs 典型点到平面 30 次），需要更多迭代
- KISS-ICP 的洞察：GM 核 + 大量迭代的组合等价于自动特征选择器

## 与点到面 ICP 对比

| 维度 | 点到点 ICP | 点到面 ICP |
|------|------------|------------|
| 残差维度 | 3D 向量 | 1D 法向距离 |
| 依赖 | 最近邻点 | 最近邻点 + 法向 |
| 收敛速度 | 通常较慢 | 通常较快 |
| 退化行为 | 不显式使用平面结构 | 平面结构强时更高效 |
| 适合 | 无法可靠估计法向、结构复杂或简化实现 | 室内/城市等局部平面明显场景 |

## 工程注意

- 点到点 ICP 仍然需要可靠初值和动态点抑制。
- 若配合 [[方法-Geman-McClure鲁棒核]]，可降低错误对应和非重叠区域影响。
- 对实时系统，通常需要体素降采样和并行化，否则迭代次数过高会成为瓶颈。
