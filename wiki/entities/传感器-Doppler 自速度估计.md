# Doppler 自速度估计

> 利用 4D 雷达每个点自带的径向多普勒速度，从静态目标点集合中鲁棒估计雷达坐标系下的 3D 自运动速度，并顺带识别动态目标。

## 定义

Doppler 自速度估计是 4D 成像雷达相对 LiDAR/相机的关键差异之一。雷达点云不仅包含三维位置，还包含目标相对雷达的径向速度。若环境中的大多数点来自静止物体，那么这些径向速度应当共同满足同一个自运动速度模型；不满足该模型的点通常来自动态目标、多径噪声或低质量测量。

在 4DRadarSLAM 中，这个模块不是后端优化因子，而是前端预处理链路：先估计雷达自速度，再把 RANSAC 内点作为静态点云用于配准和动态目标去除。

## 观测模型

对雷达坐标系下的第 `i` 个点：

- 点坐标为 `p_i = [x_i, y_i, z_i]^T`
- 距离为 `r_i = ||p_i||`
- 指向该点的单位方向为 `u_i = p_i / r_i`
- 雷达自速度为 `v_ego`
- 点的多普勒径向速度测量为 `d_i`

静止目标的理想观测关系是：

```text
d_i = -u_i^T v_ego
```

工程实现中可以把符号吸收到待估变量或测量量里。若令：

```text
H_i = [x_i/r_i, y_i/r_i, z_i/r_i]
y_i = d_i
v_r = -v_ego
```

则所有静止点组成线性系统：

```text
H v_r = y
```

最小二乘闭式解为：

```text
v_r = (H^T H)^(-1) H^T y
```

源码中对应 `v_r = (HTH).ldlt().solve(H^T * y)`，必要时也可以用 SVD 求解。注意这里不能写成 `(H^T H)(H^T y)^(-1)`：后者维度不成立，也不是最小二乘解。

## 4DRadarSLAM 流程

### 点筛选

先按距离、SNR、方位角和俯仰角过滤不可靠点。每个有效点构造 11 维特征向量，其中最后三维是归一化方向 `[x/r, y/r, z/r]`，用于构造线性观测矩阵 `H`。

### 零速度检测

系统先检查是否接近静止：取多普勒绝对值，排除 30% 最大值后取中位数。若中位数小于 `0.05 m/s`，直接认为雷达静止，输出零速度和预设协方差。这一步避免在低速或停车场景下用噪声强行拟合三维速度。

### RANSAC + 最小二乘

若不满足零速度条件，则进入鲁棒估计：

1. 每轮随机采样 5 个点。
2. 用这些点求解 `H v_r = y`。
3. 对所有候选点计算残差 `e_i = H_i v_r - y_i`。
4. 以内点数最多的速度作为候选。
5. 用所有内点重新做一次最小二乘，得到最终 `v_r`。

RANSAC 迭代次数来自：

```text
N = log(1 - p) / log(1 - (1 - outlier_ratio)^s)
```

4DRadarSLAM 配置中 `p = 0.995`、`outlier_ratio = 0.05`、采样点数 `s = 5`，对应约 115 次迭代。

### 方差估计

最终速度的不确定度用残差和正规方程矩阵近似：

```text
e = H v_r - y
C = (e^T e) (H^T H)^(-1) / (N - 3)
sigma_v = sqrt(diag(C)) + sigma_offset
```

这里 `N - 3` 对应三维速度参数的自由度扣除。若点分布退化，例如所有有效点方向过于集中，`H^T H` 会病态，速度估计尤其是横向或竖向分量会不可靠。

## 动态目标去除

RANSAC 的内点是符合静止世界速度模型的点，外点则可能来自独立运动目标或异常测量。因此 4DRadarSLAM 可以在 `enable_dynamic_object_removal` 打开时，只把 RANSAC 内点送入后续点云配准。

这个机制的价值在于：动态目标检测不依赖语义分割，也不需要跨帧跟踪目标，只利用单帧多普勒一致性就能完成第一层动静态筛选。

## 工程边界

- Doppler 自速度估计只能估计线速度，不能直接给出角速度；点云 deskew 中的角速度仍需要 IMU 或运动模型。
- 它假设静态背景占多数。若场景中大部分雷达回波来自同向运动车辆，RANSAC 可能选错模型。
- 它依赖方向向量分布。点都集中在窄视场或单一平面方向时，`H^T H` 条件数会变差。
- 它提供前端初值、动态点剔除和质量检查，但在当前 4DRadarSLAM 里不作为独立因子进入后端图优化。
