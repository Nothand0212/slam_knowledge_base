# 三角化与深度估计

> 从 2D 观测中恢复 3D 点坐标，是视觉 SLAM 中 2D-2D 跟踪通向 3D-2D PnP 的必经步骤。本文覆盖单目多视图三角化、双目立体匹配、以及双目基线对齐（立体校正）。

---

## 1. 三角化在视觉管线中的位置

```
2D-2D 特征匹配
      ↓
三角化 / 深度估计  ← 本页覆盖
      ↓
3D 地图点建立
      ↓
PnP 运动估计（3D-2D）
      ↓
后端 BA 优化
```

没有三角化，就没有 3D 点，也就无法执行 PnP 和后续的 BA 优化。

---

## 2. 单目三角化（多视图）

### 2.1 对极几何基础

已知两帧相机的相对位姿 $R, t$ 和一对匹配点 $p_1, p_2$（均以归一化坐标表示），对应的 3D 点 $P$ 满足：

$$
\begin{cases}
d_1 \cdot p_1 = P & \text{(第1帧坐标系的3D点)} \\
d_2 \cdot p_2 = R P + t & \text{(第2帧坐标系中)}
\end{cases}
$$

其中 $d_1, d_2$ 为待求解的深度值。

### 2.2 线性三角化（DLT 方法）

将两帧的投影方程联立，消去深度因子：

对第 1 帧：$p_1 \times P = 0$（叉乘产生 2 个独立方程）
对第 2 帧：$p_2 \times (R P + t) = 0$

合并为超定齐次线性方程组 $A P = 0$（4 个方程，3 个未知数）：

$$
A = \begin{bmatrix}
p_{1x} \cdot R_3^T - R_1^T \\
p_{1y} \cdot R_3^T - R_2^T \\
p_{2x} \cdot R_3^T - R_1^T \\
p_{2y} \cdot R_3^T - R_2^T
\end{bmatrix}, \quad
\text{求解 }\min_P \|A P\|, \quad \text{s.t. } \|P\| = 1
$$

SVD 分解 $A = U \Sigma V^T$，取 $V$ 的最后一列作为 $P$ 的解，再通过仿射反变换恢复实际 3D 坐标。

- **时间复杂度**：O(1)（固定大小 SVD）
- **使用项目**：ORB-SLAM3（`TwoViewReconstruction::Reconstruct`）、open_vins（初始化阶段）

### 2.3 基于逆深度的三角化

某些系统（如 open_vins 的 MSCKF 表示、SVO Pro）不直接三角化 3D 坐标，而是参数化为逆深度 $1/d$，因为：

- 逆深度对远点（$d \to \infty$）的表示比直接深度更稳定
- 高斯分布假设在逆深度空间更合理

逆深度参数化：点 $P$ 通过锚点相机位姿 $(R_a, t_a)$、归一化观测方向 $p_a$、和逆深度 $\rho = 1/d$ 表示为：

$$
P = t_a + \frac{1}{\rho} \cdot R_a p_a
$$

### 2.4 多视图三角化

使用超过 2 帧的观测可以提高三角化精度，构造更大的 $A$ 矩阵（每帧贡献 2 行），SVD 求解。

## 3. 双目三角化（立体匹配）

### 3.1 理想双目模型

简化假设：左右相机光轴平行，间距为基线 $b$（通常为 6-12 cm），水平方向有视差 $d = u_L - u_R$。

由相似三角形关系：

$$
Z = \frac{f \cdot b}{d}
$$

其中 $f$ 为焦距（像素单位），$Z$ 为深度。

**关键精度分析**——对视差做微分：

$$
\frac{\partial Z}{\partial d} = -\frac{f \cdot b}{d^2} = -\frac{Z^2}{f \cdot b}
$$

这意味着：
- 深度误差 $\propto Z^2$：距离越远，深度误差平方增长
- 深度误差 $\propto 1/b$：基线越长，误差越小
- 深度误差 $\propto 1/f$：焦距越长，误差越小

### 3.2 实际双目匹配流程

真实双目相机光轴不平行，需要通过**立体校正**（见第 4 节）对齐后再做匹配。匹配策略：

| 算法 | 匹配方法 | 搜索空间 | 验证 |
|------|---------|---------|------|
| ORB-SLAM3 (STEREO) | ORB 描述子 + 极线约束 | $L2 \pm \text{window}$ 水平条带 | 描述子距离 + ratio test |
| VINS-Fusion (STEREO) | 金字塔 KLT 光流（左图特征投影到右图） | 水平极线 ± 带宽 | 双向光流 |
| MSCKF_VIO | `stereoMatch`: 极线约束 + NCC 模板匹配 | 极线 ± 窗口 | NCC 分数 + 双向 |
| Kimera-VIO | 稀疏立体重建：GFTT 角点 + NCC 沿水平极线 | 水平极线 | NCC 峰值 + subpixel |

### 3.3 子像素视差精化

匹配到整像素位置后，通过抛物线拟合或 Lucas-Kanade 方法将视差精化到子像素精度：

- 抛物线拟合：$d_{\text{sub}} = d_0 - \frac{C_{-1} - C_{+1}}{2(C_{-1} - 2C_0 + C_{+1})}$
  其中 $C_i$ 为 NCC/ZNCC 匹配分数
- DM-VIO 的 ImmaturePoint 使用 GN 优化直接最小化沿极线的光度误差

## 4. 双目基线对齐（立体校正）

### 4.1 为什么要校正

真实双目相机的光轴不平行（存在小的旋转偏差），导致匹配搜索不是简单的水平条带，而是一条斜线。校正（Rectification）将左右图像旋转到**同一虚拟平面**，使极线变为水平共线。

### 4.2 Bouguet 算法（OpenCV 标准）

给定左右相机的旋转矩阵 $R$ 和平移向量 $t$（来自双目标定）：

1. **旋转分解**：将旋转 $R$ 对半分给左右相机：

   $$
   R_L = R^{1/2}, \quad R_R = R^{-1/2}
   $$

2. **行对齐**：构建旋转矩阵 $R_{\text{rect}}$，使基线方向（$t$ 方向）与 $X$ 轴对齐：

   $$
   \begin{aligned}
   e_1 &= \frac{t}{\|t\|} \\[4pt]
   e_2 &= \frac{(-t_y, t_x, 0)^T}{\sqrt{t_x^2 + t_y^2}} \\[4pt]
   e_3 &= e_1 \times e_2
   \end{aligned}, \quad
   R_{\text{rect}} = \begin{bmatrix} e_1^T \\ e_2^T \\ e_3^T \end{bmatrix}
   $$

3. **合成校正矩阵**：

   $$
   R_L^{\text{rect}} = R_{\text{rect}} \cdot R_L, \quad R_R^{\text{rect}} = R_{\text{rect}} \cdot R_R
   $$

4. **应用**：`cv::initUndistortRectifyMap` + `cv::remap` 生成校正后的图像对

### 4.3 各算法实现对比

| 算法 | 校正实现 | 调用位置 |
|------|---------|---------|
| ORB-SLAM3 | OpenCV `stereoRectify` + `initUndistortRectifyMap` | `Frame::UndistortKeyPoints()` |
| Kimera-VIO | 自研 `UndistorterRectifier::remap()` | 每个立体帧到达时 |
| MSCKF_VIO | OpenCV `stereoRectify` | 离线生成查找表，在线查表 |
| VINS-Fusion | （依赖外部提供校正后图像，不做在线校正） | — |

## 5. 深度不确定性建模

### 5.1 单目三角化不确定性

多视图三角化的精度取决于**基线长度**和**视差角** $\gamma$（两帧观测方向之间的夹角）：

- $\gamma$ 太小（< 1°）→ 小视差，深度高度不确定（"远点退化"）
- $\gamma$ 太大（> 30°）→ 视角变化大，特征匹配不可靠

误差协方差可通过 Fisher 信息矩阵估计：

$$
\Sigma_P \approx \left( J^T \Sigma_{uv}^{-1} J \right)^{-1}
$$

其中 $J = \frac{\partial \text{project}(P)}{\partial P}$ 为投影 Jacobian，$\Sigma_{uv}$ 为像素测量噪声（通常设为 1 px² 对角矩阵）。

### 5.2 双目深度不确定性

直接由视差公式推导：假设视差误差 $\sigma_d = 0.5 \sim 1$ px（子像素匹配精度），则：

$$
\sigma_Z \approx \frac{Z^2}{f \cdot b} \cdot \sigma_d
$$

**典型数值**：b=12cm, f=400px, Z=10m → σ_Z ≈ 21cm（Z=20m → σ_Z ≈ 83cm）

### 5.3 深度滤波器（SVO Pro 方法）

SVO Pro 不直接三角化，而是维护每个种子点的概率深度分布 $p(d | I_1, \dots, I_k)$：

- 深度建模为高斯分布 $\mathcal{N}(\mu_d, \sigma_d^2)$
- 逆深度建模为高斯 + 均匀分布的混合（处理远点/离群）
- 多帧观测通过贝叶斯更新融合 → 后验方差收敛后"成熟"为 3D 点
- 具体实现：`DepthFilter::updateSeed()`，类似 REMODE / SVO 1.0 的概率深度估计

### 5.4 DM-VIO 的 ImmaturePoint 机制

DSO/DM-VIO 的 ImmaturePoint：初始时只有光度梯度 pattern + 初始逆深度猜测 + 极大逆深度方差。
逐帧观测沿极线做 GN 优化 → 逆深度方差缩小到阈值以下 → 成熟（激活为 PointFrameResidual 参与全局优化）。

## 7. 关键设计决策

### 7.1 何时三角化

两种策略：
- **即时三角化**（ORB-SLAM3、Kimera-VIO）：匹配一旦建立立即三角化，3D 点直接参与 PnP 和 BA
- **延迟三角化**（SVO Pro、DSO）：先积累多帧观测，深度不确定性足够小后再"激活"

延迟三角化的优势：避免低质量 3D 点污染优化器；代价：新场景初始化慢。

### 7.2 深度参数化选择

| 参数化 | 适用场景 | 代表 |
|--------|---------|------|
| 直接 XYZ | 近距离、立体/RGB-D | ORB-SLAM3 |
| 逆深度 | 远点、单目 | VINS-Fusion, SVO Pro, DSO |
| Anchor + 逆深度 | 多视图、滑动窗口 | open_vins |

远点（$Z \gg b$）用 XYZ 表示时，坐标值可能为 $10^4$ 量级而误差仅 1m，优化时数值条件差；逆深度（$10^{-4}$ 量级）天然避开了这个问题。

## 8. Agent 实现提示

### 适用场景

当 Agent 需要从多帧归一化观测、双目观测或滑窗特征轨迹中生成深度初值时使用本提示。该实现通常服务于后续 PnP、SmartFactor、BA 或 MSCKF 更新，重点是把低视差、负深度和数值不稳定点挡在优化器外。

### 输入输出契约

- **输入**：同一特征的跨帧观测、每帧相机位姿 `T_w_c` 或相对投影矩阵、相机外参、最小视差/最小深度阈值、默认初始化深度。
- **输出**：`estimated_depth` 或 `Point3`、状态标记（有效/退化/需继续跟踪）、用于调试的视差角和重投影误差。
- **失败契约**：三角化深度小于阈值、SVD 解异常、观测数不足或视差过小时，不写入有效 3D 点；使用 `INIT_DEPTH` 只能作为继续优化的弱初值。

### 实现骨架（伪代码）

```python
def triangulate_track(track, poses, camera_extrinsic):
    if len(track.observations) < 2:
        return Pending("need more observations")

    anchor = track.start_frame
    A = []
    for obs in track.observations:
        P = relative_projection(anchor, obs.frame_id, poses, camera_extrinsic)
        f = normalize(obs.bearing)
        A.append(f.x * P[2] - f.z * P[0])
        A.append(f.y * P[2] - f.z * P[1])

    X_h = right_singular_vector(A)
    if near_zero(X_h.w):
        return Failure("homogeneous scale degenerate")

    depth = X_h.z / X_h.w
    if depth < min_depth:
        return Pending("depth invalid; keep default depth")
    return Success(depth)
```

```cpp
        Eigen::MatrixXd svd_A(2 * it_per_id.feature_per_frame.size(), 4);
        int svd_idx = 0;

        Eigen::Matrix<double, 3, 4> P0;
        Eigen::Vector3d t0 = Ps[imu_i] + Rs[imu_i] * tic[0];
        Eigen::Matrix3d R0 = Rs[imu_i] * ric[0];
        P0.leftCols<3>() = Eigen::Matrix3d::Identity();
        P0.rightCols<1>() = Eigen::Vector3d::Zero();

        for (auto &it_per_frame : it_per_id.feature_per_frame)
        {
            imu_j++;

            Eigen::Vector3d t1 = Ps[imu_j] + Rs[imu_j] * tic[0];
            Eigen::Matrix3d R1 = Rs[imu_j] * ric[0];
            Eigen::Vector3d t = R0.transpose() * (t1 - t0);
            Eigen::Matrix3d R = R0.transpose() * R1;
            Eigen::Matrix<double, 3, 4> P;
            P.leftCols<3>() = R.transpose();
            P.rightCols<1>() = -R.transpose() * t;
            Eigen::Vector3d f = it_per_frame.point.normalized();
            svd_A.row(svd_idx++) = f[0] * P.row(2) - f[2] * P.row(0);
            svd_A.row(svd_idx++) = f[1] * P.row(2) - f[2] * P.row(1);

            if (imu_i == imu_j)
                continue;
        }
        ROS_ASSERT(svd_idx == svd_A.rows());
        Eigen::Vector4d svd_V = Eigen::JacobiSVD<Eigen::MatrixXd>(svd_A, Eigen::ComputeThinV).matrixV().rightCols<1>();
        double svd_method = svd_V[2] / svd_V[3];
        //it_per_id->estimated_depth = -b / A;
        //it_per_id->estimated_depth = svd_V[2] / svd_V[3];

        it_per_id.estimated_depth = svd_method;
        //it_per_id->estimated_depth = INIT_DEPTH;

        if (it_per_id.estimated_depth < 0.1)
        {
            it_per_id.estimated_depth = INIT_DEPTH;
        }
```

### 实现注意事项

- 线性三角化要求坐标系一致；上例先把观测转换到锚定相机坐标系，再为每帧构造相对投影矩阵。
- `svd_V[2] / svd_V[3]` 是锚定相机下的深度，不是世界系 `Z`；写入状态前要确认参数化含义。
- `INIT_DEPTH` 不是可信测量，只能表示三角化失败后保留特征等待后续优化。
- 双目场景可先用左右相机基线给出更稳定深度，再让多视图三角化或后端优化继续精化。
