# 多传感器冗余切换（LVI-SAM 故障处理）

> 当 LiDAR 或视觉子系统出现退化时，LVI-SAM 通过健康监测 + 快速切换机制，利用另一个正常子系统的输出维持整体 SLAM 的连续运行。

## 定义

LVI-SAM 中实现的一套传感器健康度和系统状态监控体系，实时检测 LiDAR 扫描匹配退化和视觉里程计故障，在单个子系统降级时自动切换到备用方案。核心策略是 "任一传感器可用即可维持跟踪"。

## 故障模式概览

| 故障模式 | 触发场景 | LiDAR 状态 | 视觉状态 | 应对策略 |
|----------|----------|-----------|---------|---------|
| LiDAR 正常 + 视觉正常 | 特征丰富的室内外 | 正常跟踪 | 正常跟踪 | 互补融合模式 |
| LiDAR 退化 + 视觉正常 | 长走廊、隧道、开阔平面 | 部分退化 | 正常跟踪 | 视觉位姿辅助 LiDAR 初猜 |
| LiDAR 正常 + 视觉失效 | 暗光、强光、无纹理墙面 | 正常跟踪 | 失效重启 | LiDAR 独立跟踪，VINS 等待重新初始化 |
| LiDAR 退化 + 视觉失效 | 极端传感器故障 | 退化 | 失效 | IMU 单独传播（短暂维持） |
| VINS 重启 | code crash / 跟踪丢失 | 正常运行 | 重启中 | LiDAR 独立 + 为 VINS 提供初始化位姿 |

## LiDAR 健康监测

### 特征数量退化检测

LIO 子系统在进入 `scan2MapOptimization` 前检查特征点数量（`raw/codes/LVI-SAM/src/lidar_odometry/mapOptmization.cpp:L1273-L1301`）：

```cpp
void scan2MapOptimization()
{
    if (laserCloudCornerLastDSNum > edgeFeatureMinValidNum &&
        laserCloudSurfLastDSNum > surfFeatureMinValidNum)
    {
        // 正常匹配流程：KD树 + 角点/平面点优化 + LM 迭代
        kdtreeCornerFromMap->setInputCloud(laserCloudCornerFromMapDS);
        kdtreeSurfFromMap->setInputCloud(laserCloudSurfFromMapDS);
        for (int iterCount = 0; iterCount < 30; iterCount++) {
            cornerOptimization();
            surfOptimization();
            combineOptimizationCoeffs();
            if (LMOptimization(iterCount) == true) break;
        }
        transformUpdate();
    } else {
        ROS_WARN("Not enough features! Only %d edge and %d planar features available.",
                 laserCloudCornerLastDSNum, laserCloudSurfLastDSNum);
    }
}
```

阈值由 ROS 参数管理：
- `edgeFeatureMinValidNum`：默认 10（最小角特征数）
- `surfFeatureMinValidNum`：默认 100（最小面特征数）

当特征数不足时，跳过当前帧的扫描匹配，完全依赖 IMU + VINS 位姿初猜进行位姿传播。

### 6-DoF 退化检测（特征值分析）

在 LM 优化第一次迭代时检查 $H = J^T J$（6×6 Hessian）的特征值（`raw/codes/LVI-SAM/src/lidar_odometry/mapOptmization.cpp:L1221-L1249`）：

```cpp
if (iterCount == 0) {
    cv::eigen(matAtA, matE, matV);
    isDegenerate = false;
    float eignThre[6] = {100, 100, 100, 100, 100, 100};
    for (int i = 5; i >= 0; i--) {
        if (matE.at<float>(0, i) < eignThre[i]) {
            for (int j = 0; j < 6; j++)
                matV2.at<float>(i, j) = 0;
            isDegenerate = true;
        } else {
            break;
        }
    }
    matP = matV.inv() * matV2;  // 退化投影矩阵
}
```

当 `isDegenerate = true` 时，后续迭代中解出的 $\Delta \mathbf{x}$ 会被投影到非退化方向：$\Delta \mathbf{x} \leftarrow \mathbf{P} \cdot \Delta \mathbf{x}$，退化方向更新量为零。

### IMU Slerp 姿态融合

扫描匹配优化后，若 IMU 可用且 pitch 角 < 1.4 rad（避免过顶奇点），用 1% 权重的 Slerp 将 IMU roll/pitch 与 LiDAR 估计融合（`raw/codes/LVI-SAM/src/lidar_odometry/mapOptmization.cpp:L1303-L1331`）：

```cpp
if (std::abs(cloudInfo.imuPitchInit) < 1.4) {
    double imuWeight = 0.01;
    transformQuaternion.setRPY(transformTobeMapped[0], 0, 0);
    imuQuaternion.setRPY(cloudInfo.imuRollInit, 0, 0);
    tf::Matrix3x3(transformQuaternion.slerp(imuQuaternion, imuWeight))
        .getRPY(rollMid, pitchMid, yawMid);
    transformTobeMapped[0] = rollMid;
    // pitch 同理
}
```

这为 LiDAR 退化时的 roll/pitch 提供弱约束，防止完全依赖退化方向的不确定估计。

## 视觉健康监测

### VINS 失效检测

VIS 子系统每帧优化后执行 `failureDetection()` 检查：

- `solver_flag == INITIAL` 超过一定时间未初始化成功 → 视为初始化失败
- 非线性优化中如果残差或状态量超出合理范围 → 触发 `failure_occur = 1`

触发后执行完整重启（`raw/codes/LVI-SAM/src/visual_odometry/visual_estimator/estimator.cpp:L191-L199`）：

```cpp
if (failureDetection()) {
    ROS_ERROR("VINS failure detection!");
    failure_occur = 1;
    clearState();       // 清空所有状态
    setParameter();     // 重新设置参数
    ROS_ERROR("VINS system reboot!");
    return;
}
```

`clearState()` 会重置窗口内所有位姿、速度、偏置、预积分和特征点数据（`raw/codes/LVI-SAM/src/visual_odometry/visual_estimator/estimator.cpp:L22-L80`）。重置后 VIS 重新进入 INITIAL 状态，等待 LiDAR 提供新初始位姿。

### VINS 重置通知

VIS 重启后通过 `failure_occur` 标志位将 `odomResetId` 递增，嵌入在发送给 LIO 的 odometry 消息中（`raw/codes/LVI-SAM/src/lidar_odometry/imuPreintegration.cpp:L355-L393`）：

```cpp
odometry.pose.covariance[0] = double(imuPreintegrationResetId);
odometry.pose.covariance[1] = prevBiasOdom.accelerometer().x();
// ... bias info embedded in covariance fields
```

LIO 侧收到新的 `odomResetId` 后，丢弃当前 VINS 位姿，重置 IMU 预积分参数（`raw/codes/LVI-SAM/src/lidar_odometry/imuPreintegration.cpp:L144-L149`）：

```cpp
if (currentResetId != imuPreintegrationResetId) {
    resetParams();
    imuPreintegrationResetId = currentResetId;
    return;
}
```

`resetParams()` 重置系统初始化标志和 IMU 预积分状态，等待新的一致性数据。

## 位姿初猜切换逻辑

`updateInitialGuess()` 是子系统冗余切换的核心（`raw/codes/LVI-SAM/src/lidar_odometry/mapOptmization.cpp:L801-L868`）：

```pseudo
function updateInitialGuess():
    if cloudKeyPoses3D is empty:
        // 系统初始化
        set transform from IMU attitude (roll/pitch/yaw)

    elif cloudInfo.odomAvailable AND odomResetId matches:
        // 优先级 1: VINS 位姿增量作为初猜
        compute VINS incremental pose
        apply to current transform estimate
        return

    else:
        // VINS 不可用 → 标记失效
        lastVinsTransAvailable = false
        odomResetId = cloudInfo.odomResetId

    if cloudInfo.imuAvailable:
        // 优先级 2: 纯 IMU 角增量作为初猜（仅旋转）
        compute IMU angular increment
        apply rotation increment
```

决策树：

```
                 ┌─ odomAvailable? ──── YES ── odomResetId 一致? ── YES ── [VINS 位姿初猜]
系统已初始化 ───┤
                 └─ VINS 不可用 ── imuAvailable? ── YES ── [IMU 角增量初猜]
                                                                NO  ── [保持上一帧位姿]
```

## IMU 预积分故障检测

LIO 的 IMU 预积分优化完成后，执行 `failureDetection()`（`raw/codes/LVI-SAM/src/lidar_odometry/imuPreintegration.cpp:L317-L335`）：

```cpp
bool failureDetection(const gtsam::Vector3& velCur,
                      const gtsam::imuBias::ConstantBias& biasCur)
{
    Eigen::Vector3f vel(velCur.x(), velCur.y(), velCur.z());
    if (vel.norm() > 30) {           // 速度超过 30 m/s → 异常
        ROS_WARN("Large velocity, reset IMU-preintegration!");
        return true;
    }

    Eigen::Vector3f ba(...), bg(...);
    if (ba.norm() > 0.1 || bg.norm() > 0.1) {  // 偏置过大 → 异常
        ROS_WARN("Large bias, reset IMU-preintegration!");
        return true;
    }
    return false;
}
```

触发后调用 `resetParams()`，清空 IMU 预积分状态并重新初始化。

## 视觉特征健康度

特征跟踪器的冗余体现在 `addPoints()` 和 `rejectWithF()`：

- `rejectWithF()`（`raw/codes/LVI-SAM/src/visual_odometry/visual_feature/feature_tracker.cpp:L169-L202`）：使用 RANSAC + 基础矩阵剔除异常匹配
- `setMask()` + `addPoints()`（feature_tracker.cpp:L71-L79）：当 `forw_pts.size() < MAX_CNT` 时自动补充新 `goodFeaturesToTrack` 检测的特征点，保持恒定特征数量
- `track_cnt` 计数：跟踪时间长的特征优先保留（`cnt_pts_id` 排序）

## Agent 实现提示

### 适用场景
- 室外 SLAM 中传感器可能因环境变化而发生部分失效
- 需要实现多传感器健康度监控和优雅降级
- 设计多传感器融合系统时的故障切换参考

### 输入输出契约
- **输入**：LiDAR 特征点云（角点+面点）、IMU 数据（加速度+角速度）、视觉里程计 odometry 消息
- **输出**：位姿估计（6-DoF）、故障标志位、健康状态诊断
- **坐标/单位**：角度 rad，平移 m，速度 m/s，偏置 acc: m/s² / gyr: rad/s

### 实现骨架（伪代码）
```pseudo
class SensorHealthMonitor:
    def __init__(self):
        self.lidar_healthy = True
        self.vis_healthy = False  # 需要初始化
        self.vis_reset_id = -1

    def check_lidar_health(lidar_features, hessian):
        n_corner = count_corner_features(lidar_features)
        n_surf = count_surface_features(lidar_features)
        if n_corner < MIN_CORNER or n_surf < MIN_SURF:
            return DEGRADED

        # 特征值退化检测
        eigen_vals = eigen_decompose(hessian)
        degenerate_dims = [i for i, v in enumerate(eigen_vals) if v < THRESHOLD]
        if degenerate_dims:
            return PARTIALLY_DEGRADED(degenerate_dims, eigen_vals)
        return HEALTHY

    def check_vis_health(vis_optimizer_status):
        if vis_optimizer_status == FAILED:
            return FAILED
        if vis_optimizer_status == INITIAL and time_since_start > TIMEOUT:
            return INIT_FAILED
        return HEALTHY

    def select_initial_guess(lidar_health, vis_health, vis_odom, imu_data):
        if vis_health == HEALTHY:
            return VIS_INCREMENTAL_GUESS(vis_odom)
        elif imu_data.available:
            return IMU_ATTITUDE_GUESS(imu_data)
        else:
            return HOLD_PREVIOUS_POSE()

    def handle_vis_failure():
        vis_optimizer.clearState()
        vis_optimizer.setParameter()
        vis_reset_id += 1
        notify_lio(vis_reset_id)
```

### 实现注意事项
- `odomResetId` 通过 `odometry.pose.covariance[0]` 传递，这是一个非常规用法 —— covariance 矩阵的 `[0]` 元素被覆写为 reset ID 的整数值。实现时需确保接收端正确解析
- VIS 重新初始化时依赖 LiDAR 提供 `Ps[i], Rs[i], Vs[i], Ba[i], Bg[i]`，如果 LIO 也正在退化，VIS 初始化可能失败
- 偏置阈值（0.1 for acc, 0.1 for gyr）和速度阈值（30 m/s）是经验值，需要根据 IMU 传感器规格调整
- `clearState()` 中直接 `delete` 预积分对象并置 `nullptr`，可能导致正在使用的 IMU 数据丢失 —— 要确保线程安全
- 特征值退化检测的阈值 `eignThre = [100, 100, 100, 100, 100, 100]` 针对 6 个自由度一视同仁；更精细的实现应如 SuperOdom 那样为每个自由度设定不同阈值
