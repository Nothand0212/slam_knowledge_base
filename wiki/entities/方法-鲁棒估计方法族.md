# 鲁棒估计方法族

> SLAM/多传感器融合中对抗观测粗差的完整武器库：从单核函数到自适应阈值再到两轮优化 + 卡方剔除的递进方案。

## 概述

### 最小二乘的脆弱性

标准非线性最小二乘最小化残差平方和：

$$\min_{\mathbf{x}} \sum_i \|r_i(\mathbf{x})\|^2$$

当残差服从零均值正态分布时，这个 estimator 是极大似然估计（MLE）且统计最优。但 SLAM 真实场景中大量观测是**粗差**（outlier）：

- GNSS 多径、NLOS 位置误差可达 10–50m
- 视觉误匹配/运动模糊产生的重投影误差
- LiDAR 中动态物体、非重叠区域产生的错误对应点

这些粗差的分布根本不是正态的。一个 50m 的 GNSS 粗差在最小二乘中贡献 $50^2=2500$ 的代价，会轻易主导整个优化，把状态估计拖离真值。

### 鲁棒估计的本质

鲁棒估计的目标是：**当数据中大部分观测是内点时，给出近似最优估计；当部分数据是粗差时，估计结果不应被破坏**。

实现的两种基本策略：
1. **M-estimator（鲁棒核函数）**：在最小二乘中引入重投影函数 $\rho(r)$，限制每个残差的贡献上限
2. **硬阈值剔除**：用统计检验（如卡方检验）判定粗差后直接移除/重赋权

实践中两者往往**组合使用**：先用 (1) 保护优化不偏离，再用 (2) 识别并清除粗差，最后用纯最小二乘精调。

## 方法对比表

### 关键概念说明

- **影响函数 $\psi(e) = \partial\rho/\partial e$**：测量残差变化对总损失的影响。L2 的 $\psi(e)=e$ 无界，一个粗差可产生无限大的梯度。
- **权重函数 $w(e) = \psi(e)/e$**：将鲁棒估计理解为迭代重加权最小二乘（IRLS）。$w(e)=1$ 即为普通 L2。
- **断点（breakdown point）**：估计量能容忍的粗差比例。L2 断点为 0，即一个粗差就破坏结果。

## 各方法详解

### Huber Loss

**公式**：

$$\rho(e) = \begin{cases}
\frac{1}{2}e^2 & |e| \leq \delta \\[4pt]
\delta|e| - \frac{1}{2}\delta^2 & |e| > \delta
\end{cases}$$

**权重函数**：$w(e) = \min(1, \delta/|e|)$。小残差保持 L2 权重 1，大残差权重反比于残差幅值。

**影响曲线分析**：$\psi(e) = \max(-\delta, \min(e, \delta))$ — 梯度被截断在 $\pm\delta$。这是唯一**凸**的鲁棒核函数，保证了优化问题的全局收敛性。

**使用建议**：
- **适用**：通用非线性优化，粗差比例不高（<20%）的场景。凸性是最大优势。
- **不适用**：大量大粗差的场景——权重衰减太慢（$O(1/|e|)$），极大粗差仍能产生可观影响。
- **典型 $\delta$ 值**：GNSS 位置残差 $\delta=1.0$（m）；视觉重投影 $\delta=1.345$（像素）

**Ceres 使用**：
```cpp
ceres::LossFunction* loss = new ceres::HuberLoss(1.0);
problem.AddResidualBlock(cost_function, loss, parameters);
```
- GTSAM: `Huber::Create(k)` 对应 $\delta = k\sqrt{2}$（注意 GTSAM 参数化差异）

### Cauchy Loss

**公式**：

$$\rho(e) = \frac{c^2}{2} \log\left(1 + \frac{e^2}{c^2}\right)$$

**权重函数**：$w(e) = \frac{1}{1 + e^2/c^2}$。权重下降比 Huber 快（$O(1/e^2)$），但永远不为零。

**影响曲线分析**：$\psi(e) = \frac{e}{1+e^2/c^2}$ — 当 $|e| \to \infty$ 时 $\psi(e) \to 0$，即极大粗差不会产生梯度。比 Huber 对外点更鲁棒，但对小残差的惩罚比 Huber 略轻。

**使用建议**：
- **适用**：LiDAR/视觉残差鲁棒估计，大残差较多但又不希望完全拒识的场景
- $c$ 取值影响很大：$c$ 太小 → 几乎所有数据被降权；$c$ 太大 → 接近 L2

**Ceres 使用**：
```cpp
ceres::LossFunction* loss = new ceres::CauchyLoss(0.5);
problem.AddResidualBlock(cost_function, loss, parameters);
```

### Geman-McClure Loss

**公式**：

$$\rho(e) = \frac{e^2/2}{\sigma + e^2}$$

**权重函数**：$w(e) = \frac{\sigma^2}{(\sigma + e^2)^2}$。权重按 $O(1/e^4)$ 衰减，是常用鲁棒核中最激进的大残差抑制。

**影响曲线分析**：$\psi(e) = \frac{e\sigma}{(\sigma+e^2)^2}$。残差超过 $\sigma$ 后梯度迅速归零，等价于**几乎完全拒识**大残差。

**对比 Huber、Cauchy**：当 $e^2 \gg \sigma$ 时，Huber 权重 $\sim \sigma/|e|$，Cauchy 权重 $\sim c^2/e^2$，GM 权重 $\sim \sigma^2/e^4$。GM 的抑制强度远超前两者，代价是需要更多优化迭代。

**典型应用 — KISS-ICP**：用 GM 核对所有 ICP 残差加权，实现**隐式特征选择**——不提取 edge/planar 特征，而是靠 GM 核赋予高残差点低权重来自动过滤动态物体和错误对应。此时 $\sigma$ 由自适应阈值模块在线学习。

**使用建议**：
- **适用**：ICP 中大量错误对应或动态点、少先验依赖鲁棒优化筛选的场景
- **不适用**：系统已有语义动态剔除、严格几何验证时，过强鲁棒核降低有效约束数量
- KISS-ICP 默认 500 次迭代（相比 Voxelized GICP 的 20–50 次显著更多）

**Ceres 使用**：Ceres 未内置 Geman-McClure Loss，需自定义 `LossFunction`：
```cpp
class GemanMcClureLoss : public ceres::LossFunction {
public:
    explicit GemanMcClureLoss(double sigma) : sigma_(sigma) {}
    void Evaluate(double s, double rho[3]) const override {
        // s = e^2 (squared residual)
        const double denom = sigma_ + s;
        rho[0] = 0.5 * s / denom;              // ρ(e)
        rho[1] = sigma_ / (denom * denom);     // ρ'(e), ρ' = σ/(σ+s)²
        rho[2] = -2.0 * sigma_ / (denom * denom * denom); // ρ''(e)
    }
private:
    double sigma_;
};
```

### Tukey Biweight Loss

**公式**：

$$\rho(e) = \begin{cases}
\frac{c^2}{6}\left[1 - \left(1 - \frac{e^2}{c^2}\right)^3\right] & |e| \leq c \\[4pt]
\frac{c^2}{6} & |e| > c
\end{cases}$$

**权重函数**：$w(e) = \max(0, (1 - e^2/c^2)^2)$。核心特点：$|e| > c$ 时权重**恰好为零**，属于**硬拒绝**类鲁棒核。

**影响曲线分析**：$\psi(e)$ 在 $|e| \leq c$ 内平滑上升再下降，在 $|e|=c$ 处达到零且一阶连续。这是唯一同时具备 (a) 完全拒识外点 和 (b) 平滑过渡的鲁棒核函数。缺点是**完全非凸**，初始值差时优化容易陷入局部极小。

**使用建议**：
- **适用**：粗差比例可估计（通常 <50%），且已有较好初始值的场景。常用于 Bundle Adjustment 的最终精调轮次。
- **不适用**：初始状态很差或粗差比例超过 50% 时，优化会卡在局部极小。
- **参数 $c$ 选择**：通常取 $c = 4.685\sigma$（对应的 L2 等价效率为 95%）

**Ceres 使用**：
```cpp
ceres::LossFunction* loss = new ceres::TukeyLoss(10.0);
problem.AddResidualBlock(cost_function, loss, parameters);
```

### 自适应阈值

**来源**：[[算法-KISS-ICP]] 的核心机制。

**核心思路**：不再手工设定鲁棒核的尺度参数，而是从运动模型预测偏差中**在线估计**。由 `AdaptiveThreshold::UpdateModelDeviation()` 实现：

$$e_{\text{model}} = |\Delta\mathbf{t}| + 2 \cdot r_{\max} \cdot \sin\left(\frac{|\Delta\mathbf{R}|}{2}\right)$$

这是相邻两帧点云之间点可能发生的最大位移估计。若 $e_{\text{model}} > \text{min\_motion\_th}$（0.1m），则将 $e_{\text{model}}^2$ 累加到 SSE，在线计算：

$$\sigma = \sqrt{\frac{\text{SSE}}{N}}$$

初始 $\sigma=2.0$m，随运动逐渐收敛到真实误差水平。

**双重用途**：
- $3.0 \cdot \sigma$ 作为 ICP correspondence 截断半径（3σ 正态截断）
- $\sigma$ 作为 Geman-McClure 核的 kernel_scale

**设计意义**：自适应阈值把"ICP 应该接受多远的对应点"从手工调参变成在线估计。运动模型预测越不准 → 阈值越大 → 允许更宽的匹配搜索；运动稳定后 → 阈值收缩 → 减少错误对应。

**工程边界**：
- 静止时不更新（误差 < 0.1m），避免噪声污染尺度估计
- 场景退化或动态物体过多时 $\sigma$ 可能被错误增大
- 该机制适合少先验、靠鲁棒优化自适应筛选的配准系统

**伪代码骨架**：
```python
def adaptive_threshold_update(odom_delta, motion_th=0.1):
    e = |Δtranslation| + 2 * max_range * sin(|Δrotation| / 2)
    if e > motion_th:
        SSE += e²
        N_samples += 1
        sigma = sqrt(SSE / N_samples)
    return sigma
```

### 两轮优化 + 卡方粗差剔除

**来源**：[[算法-IC-GVINS]]、[[算法-OB_GINS]] 的滑动窗口优化策略。

**核心思路**：先用鲁棒核保守收敛，再用卡方检验识别每个因子的统计异常度，剔差后精调。

#### 迭代比例分配

```
Round 1（~25% 迭代）：HuberLoss → 保守收敛
          ↓
    卡方检验粗差检测（GNSS + 视觉因子）
          ↓
    重建 Problem（GNSS 重赋权/不加核，视觉粗差移除）
          ↓
Round 2（~75% 迭代）：无核函数 → 精细收敛
```

**设计依据**：第一轮只用少量迭代，因为含粗差时 Hessian 近似不可靠；第二轮用大部分迭代获取理论最优解。

#### 标准残差与卡方检验

Ceres 的 `EvaluateResidualBlock` 返回 `cost = 1/2 * ||r||²`，构造检验统计量：

$$\chi^2 = 2 \cdot \text{cost} = \|\mathbf{r}\|^2$$

在正态假设下 $\|\mathbf{r}\|_{\Omega}^2 \sim \chi^2_m$（$m$ 为残差自由度，$\Omega$ 为信息矩阵）。

**常用阈值**（$\alpha=0.05$）：

| 自由度 $m$ | $\chi^2$ 阈值 | 对应观测 | 项目 |
|-----------|---------------|---------|------|
| 1 | 3.841 | 点到面距离 | |
| 2 | 5.991 | 重投影误差 | IC-GVINS 视觉粗差 |
| 3 | 7.815 | GNSS 位置残差 | IC-GVINS, OB_GINS |

#### 重赋权 vs 删除

| 策略 | 方法 | 适用场景 |
|------|------|---------|
| **重赋权** | `std *= sqrt(χ² / threshold)` | GNSS 粗差：信号退化而非完全错误 |
| **删除** | `RemoveResidualBlock(id)` | 视觉重投影：误匹配应完全移除 |

重赋权的物理含义：若卡方统计量是阈值的 $k$ 倍，将噪声标准差扩大 $\sqrt{k}$ 倍，等效于降低该观测在优化中的权重。

#### 关键代码片段

GNSS 粗差检测（IC-GVINS，`raw/codes/IC-GVINS/ic_gvins/ic_gvins/ic_gvins.cc:L1241-L1267`）：

```cpp
double chi2_threshold = 7.815;  // χ²₃(0.05)
for (auto &block : residual_block) {
    double cost;
    problem.EvaluateResidualBlock(block.first, false, &cost, nullptr, nullptr);
    double chi2 = cost * 2;
    if (chi2 > chi2_threshold) {
        double scale = sqrt(chi2 / chi2_threshold);
        block.second->std *= scale;  // 重赋权
        outliers_counts++;
    }
}
```

视觉粗差剔除（IC-GVINS，`raw/codes/IC-GVINS/ic_gvins/ic_gvins/ic_gvins.cc:L1269-L1297`）：

```cpp
// 先收集再删除（遍历中删除会破坏内部索引）
vector<ResidualBlockId> outlier_ids;
for (auto &id : residual_ids) {
    double cost;
    problem.EvaluateResidualBlock(id, false, &cost, nullptr, nullptr);
    if (cost * 2.0 > 5.991)  // χ²₂(0.05)
        outlier_ids.push_back(id);
}
for (auto &id : outlier_ids) {
    problem.RemoveResidualBlock(id);  // 批量删除
}
```

#### 为什么 Round 2 去掉核函数

粗差已被统计检验识别并处理（重赋权或删除），剩余观测近似服从正态分布。此时普通最小二乘就是 MLE，加上核函数反而会降低有效信息量。GNSS 因子虽不加 Huber，但信息矩阵已在 Round 1 检测中通过 `std *= scale` 衰减，等价于软加权。

## 选型指南

### 按场景选择

```
含少量粗差（<20%）→ HuberLoss(δ)
    │
    ├─ 需保证全局凸性 → Huber
    ├─ 粗差更多但不想完全拒识 → CauchyLoss(c)
    └─ 有大量异常 → 
        │
        ├─ ICP 配准，少先验 → GM 核 + 自适应阈值（KISS-ICP 模式）
        ├─ Bundle Adjustment → TukeyLoss(c)（硬拒绝，需好初值）
        └─ GNSS+视觉融合，需要严格统计判定 → 两轮优化 + 卡方检验
```

### 按阶段选择

| 优化阶段 | 推荐方法 | 原因 |
|---------|---------|------|
| 全局初始化（Gravity、IMU bias） | Huber | 凸性保证收敛到正确盆 |
| 滑动窗口粗优化 | Huber（Round 1） | 含粗差时 Hessian 不可靠，迭代要少 |
| 粗差检测 | 卡方检验（$\alpha=0.05$） | 统计判定 > 启发式阈值 |
| 滑动窗口精调 | 无核函数（Round 2） | 粗差已剔除，纯 L2 是最优的 |
| 最终 BA | Tukey | 已完成初步收敛，用硬拒绝获取干净内点集 |
| LiDAR scan matching | GM + 自适应阈值 | 自动适应场景几何复杂度 |

### 核函数梯度衰减速度参考

| 核函数 | 权重 $w(e)$ 渐近行为（$|e| \to \infty$） | 抑制强度 |
|--------|------|---------|
| Huber | $\sim \delta/|e|$ | 弱 |
| Cauchy | $\sim c^2/e^2$ | 中 |
| GM | $\sim \sigma^2/e^4$ | 强 |
| Tukey | $=0$（$|e|>c$ 时） | 完全拒识 |

### 反模式（避免）

- **多个鲁棒核叠加使用**：每个残差块一个 LossFunction 即可，不要在图优化中叠加多个核函数
- **Tukey 用于初始化**：非凸 + 硬拒绝极易卡在局部极小
- **卡方阈值使用标准 $\alpha=0.05$ 当残差非正态**：多径或误匹配呈强非高斯时，固定阈值会漏检
- **静态场景用自适应阈值**：静止时 sigma 不更新，初始值 2.0m 维持过宽，失去筛选作用
- **两轮优化中 Round 1 用 Cauchy/Tukey**：Round 1 目的是保守收敛，Huber 的凸性是首要考量

## Agent 实现提示

### 适用场景

- **应该使用**：任何含观测粗差的非线性最小二乘优化（SLAM 后端、BA、scan matching、GNSS 融合）
- **多层组合**：粗差比例未知 → 用 Huber（凸保证）; 粗差分布已知 → 用卡方检验；已知好初值 → 用 Tukey
- **不应该使用**：所有观测已被前端严格几何验证（如 RANSAC 后）、粗差比例极低（<1%）时纯 L2 更优

### 输入输出契约

- **输入**：
  - Ceres Problem 对象 + 残差块列表（含 `ResidualBlockId` 和观测数据指针）
  - 鲁棒核类型与参数：Huber($\delta$), Cauchy($c$), Tukey($c$), 自定义 GM($\sigma$)
  - 卡方检验阈值：自由度 $m$ × 置信度 $\alpha$（常用 0.05）
  - 优化配置：第一轮/第二轮迭代数分配
- **输出**：
  - 优化后的状态参数
  - 粗差标记：哪些因子被剔除、哪些被重赋权
  - 可选：每轮优化前/后的残差分布诊断
- **坐标/单位**：信息矩阵的编码决定卡方统计量的量纲；确保残差量与信息矩阵单位一致

### 实现骨架（伪代码）

```python
def robust_optimization(problem, factors, config):
    # === Step 1: 选择并设置核函数 ===
    if config.loss_type == "Huber":
        loss = HuberLoss(1.0)
    elif config.loss_type == "Cauchy":
        loss = CauchyLoss(0.5)
    elif config.loss_type == "GM":
        loss = GemanMcClureLoss(sigma=config.adaptive_sigma)
    elif config.loss_type == "Tukey":
        loss = TukeyLoss(4.685 * config.estimated_noise_std)

    # === Step 2: Round 1 — 带核函数的保守收敛 ===
    for factor in factors.all():
        factor.set_loss_function(loss)
    solve(problem, max_iter=config.total_iters // 4)

    # === Step 3: 卡方粗差检测 ===
    outliers = []
    for factor in factors:
        cost = problem.EvaluateResidualBlock(factor.id).cost
        chi2 = cost * 2  # Ceres cost = 0.5 * ||r||²
        threshold = chi2_table[dof=factor.dof, alpha=0.05]
        if chi2 > threshold:
            if factor.cull_strategy == "reweight":
                factor.info *= threshold / chi2  # 等效降低信息权重
            elif factor.cull_strategy == "remove":
                outliers.append(factor.id)

    # === Step 4: 重建 Problem ===
    for factor in factors.gnss:
        problem.RemoveResidualBlock(factor.id)
    re_add_gnss_factors(problem, use_loss_function=False)  # 无核

    for oid in outliers:
        problem.RemoveResidualBlock(oid)  # 批量删除视觉粗差

    # === Step 5: Round 2 — 纯最小二乘精调 ===
    solve(problem, max_iter=config.total_iters * 3 // 4)

    return state, outlier_report
```

### 实现注意事项

1. **不要在第一轮过程中删除 ResidualBlock**：`RemoveResidualBlock` 会改变内部索引，必须先收集所有 outlier ID 再统一删除
2. **启动 `enable_fast_removal`**：`ceres::Problem::Options options; options.enable_fast_removal = true;` 可显著加速删除
3. **Ceres cost 乘 2 才是卡方**：`cost = 0.5 * ||r||²`，因此 $\chi^2 = 2 \times \text{cost}$
4. **卡方阈值依赖正确建模的信息矩阵**：若 $\Omega$ 未正确编码残差的真实协方差，卡方检验无效
5. **自适应阈值静止时不更新**：防止噪声污染尺度估计；但启动阶段需要合理初始值（如 2.0m）
6. **GTSAM 的 Huber 参数化不同**：GTSAM 使用 `Huber::Create(k)`，其 $\delta = k\sqrt{2}$，与 Ceres 的 `HuberLoss(δ)` 差 $\sqrt{2}$ 倍
7. **Round 1 只用 ~25% 迭代**：含粗差时 Hessian 不可靠，过多迭代既耗时又可能偏离正确盆地
