# 连续时间线性插值

> 用帧起始和结束两个位姿描述扫描期间的连续轨迹：平移线性插值，旋转用 slerp，在复杂度和去畸变效果之间折中。

## 定义

连续时间线性插值是 CT-ICP 对 LiDAR 帧内轨迹的简化连续表示。它不把整帧点云绑定到单一位姿，而是让每个点按自己的归一化时间戳 `α ∈ [0,1]` 取得一个插值位姿。

## 插值模型

平移部分：

```text
t(α) = (1 - α) t_begin + α t_end
```

旋转部分：

```text
q(α) = slerp(q_begin, q_end, α)
```

于是每个点 `p_i` 使用 `T(α_i)` 变换，而不是统一使用帧起始或帧结束位姿。

## 为什么不用更复杂模型

| 模型 | 状态量 | 优点 | 代价 |
|------|--------|------|------|
| 单位姿 | 1 个 pose | 最简单 | 无法表达扫描畸变 |
| begin/end 线性插值 | 2 个 pose | 变量少，可表达一阶帧内运动 | 不能表达强加速度/高阶运动 |
| B-spline | 多个控制点 | 平滑、表达力强 | 实现和边界处理更复杂 |
| Gaussian Process 轨迹 | 多状态点和核函数 | 理论优雅 | 计算和建模成本高 |

CT-ICP 选择 begin/end 线性插值，是因为机械式 LiDAR 的 10Hz 扫描周期内，一阶近似通常能显著改善畸变，同时仍保持实时。

## 工程注意

- 每个点必须保留帧内相对时间，否则无法插值。
- 旋转不能用欧拉角线性插值，应该用四元数 slerp 或 Lie 群插值。
- 高动态平台上，begin/end 两点模型可能不足，需要 IMU 或更高阶连续时间轨迹。

## Agent 实现提示

### 适用场景

LiDAR 扫描去畸变（deskew）、连续时间轨迹优化、帧内运动补偿。适合帧周期内平台有明显运动但不能或不想引入 IMU 的场景；需要在位姿流形上按归一化时间 `α∈[0,1]` 对每个点独立查询插值位姿。

### 输入输出契约

- 输入：帧起始位姿 `begin_pose`（含时间戳 `dest_timestamp`），帧结束位姿 `end_pose`（含时间戳 `dest_timestamp`），目标时间戳 `timestamp` 或归一化 alpha `∈[0,1]`，目标 frame_id（可选，默认取 begin 的 ref_frame_id）。
- 输出：插值位姿 `TPose`（含旋转四元数、平移向量、插值时间戳、坐标系 ID）。
- 前置条件：`begin_pose` 和 `end_pose` 在同一参考坐标系下；`begin_pose.dest_timestamp ≤ timestamp ≤ end_pose.dest_timestamp` 或 alpha 在 [0,1]。
- 坐标约定：平移在世界系（ref_frame）中线性插值 `t = (1-α)*t_begin + α*t_end`；旋转用四元数 slerp；插值位姿仍表达在原参考坐标系下。

### 实现骨架（伪代码）

```text
function InterpolatePose(begin_pose, end_pose, timestamp):
    alpha = (timestamp - begin_pose.timestamp) / (end_pose.timestamp - begin_pose.timestamp)
    alpha = clamp(alpha, 0, 1)

    // TSE3 插值（平移线性 + 旋转 slerp）
    interpolated.pose.quat = slerp(begin_pose.quat, end_pose.quat, alpha)
    interpolated.pose.tr   = (1 - alpha) * begin_pose.tr + alpha * end_pose.tr
    interpolated.timestamp = (1 - alpha) * begin_pose.timestamp + alpha * end_pose.timestamp
    interpolated.ref_frame_id = begin_pose.ref_frame_id
    interpolated.dest_frame_id = begin_pose.dest_frame_id

    return interpolated
```

```cpp
inline T GetAlphaTimestamp(T mid_timestamp, const TPose<T> &other_pose) const {
    T min_timestamp = std::min(dest_timestamp, other_pose.dest_timestamp);
    T max_timestamp = std::max(dest_timestamp, other_pose.dest_timestamp);
    if (min_timestamp > mid_timestamp) { /* clip to 0 */ return T(0); }
    if (max_timestamp < mid_timestamp) { /* clip to 1 */ return T(0); }
    if (min_timestamp == max_timestamp)
        return T(1.0);
    return (mid_timestamp - min_timestamp) / (max_timestamp - min_timestamp);
}
```

TSE3<T> TSE3<T>::Interpolate(const TSE3<T> &rhs, T weight) const {
    TSE3<T> result;
    result.quat = quat.slerp(weight, rhs.quat);           // 旋转 slerp
    result.tr = (T(1) - weight) * tr + weight * rhs.tr;   // 平移线性插值
    return result;
}
```

TPose<T> TPose<T>::InterpolatePose(const TPose<T> &other_pose, T timestamp,
                                    frame_id_t new_dest_frame_id) const {
    CHECK(dest_timestamp <= timestamp && timestamp <= other_pose.dest_timestamp)
        << "The timestamp cannot be interpolated between the two poses" << std::endl;
    CHECK (other_pose.ref_frame_id == ref_frame_id)
        << "Invalid operation: Cannot interpolate two frames not expressed in the same reference frame.";
    TPose<T> new_pose;
    new_pose.ref_frame_id = ref_frame_id;
    new_pose.dest_frame_id = dest_frame_id == other_pose.ref_frame_id ? dest_frame_id : new_dest_frame_id;
    new_pose.ref_timestamp = ref_timestamp;
    new_pose.dest_timestamp = timestamp;
    new_pose.pose = pose.Interpolate(other_pose.pose, GetAlphaTimestamp(timestamp, other_pose));
    return new_pose;
}
```

### 实现注意事项

- 旋转插值必须用 slerp 或 Lie 群对数/指数映射插值（`Log(q₂*q₁⁻¹)*α` 再 Exp），禁止对欧拉角或四元数分量线性插值（结果不是单位四元数）。
- CT-ICP 中使用 `begin_pose.InterpolatePoseAlpha(end_pose, alpha)` 为每个 LiDAR 点获取独立位姿，然后去畸变：`point_world = interpolated_pose * T_cl * raw_point`。
- 帧内相对时间戳 `α` 的精度取决于 LiDAR 硬件扫描速率校准；OS1-64 等旋转式 LiDAR 的 `α` 通常按方位角归一化。
- `GetAlphaTimestamp` 在时间戳超出范围时会钳位到 0 或 1（而非报错），实际使用中应确保输入时间戳在帧区间内。
- 插值模型的两个位姿（begin/end）同时作为 Ceres 优化变量时，需要提供解析 Jacobian（pose 插值对各 Pose 的偏导），详见 CT-ICP 的 `cost_functions.h`。
