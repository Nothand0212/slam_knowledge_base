# 多会话坐标系对齐

> 引入 anchor 位姿节点，将多个独立 SLAM 会话的局部轨迹变换到统一中心坐标系下进行联合优化。

## 问题描述

多个 SLAM 会话（sessions）各自运行在独立的局部坐标系中：
- Session 1 以起始位置为原点建立坐标系 $C_1$
- Session 2 以自身起始位置为原点建立坐标系 $C_2$
- ...
- Session N 以自身起始位置为原点建立坐标系 $C_N$

每个会话内部已有通过里程计约束和内部回环构建的局部位姿图，但会话之间没有直接关联。任务是将所有轨迹对齐到一个统一的中心坐标系 $C_{\text{central}}$。

### 输入输出

- **输入**：N 个 session 的位姿图文件（g2o 格式），每个包含 VERTEX_SE3:QUAT 节点和 EDGE_SE3:QUAT 边
- **输出**：所有轨迹统一在中心坐标系下的优化后位姿，以及每个 session 的 anchor 变换矩阵

## 核心思想：Anchor 节点

### 概念

为每个 session $i$ 引入一个 **anchor 位姿** $T_a^i \in SE(3)$，使得任意帧 $k$ 的全局位姿可以通过：

$$T_g^{i,k} = T_a^i \cdot T_l^{i,k}$$

从局部位姿 $T_l^{i,k}$ 变换得到，其中 $\cdot$ 表示 SE(3) 复合（$\oplus$ 运算）。

### 节点编号方案

```cpp
// kSessionStartIdxOffset = 1000000（百万级间隔）
int genGlobalNodeIdx(session_idx, node_offset):
    return session_idx * 1000000 + node_offset + 1

int genAnchorNodeIdx(session_idx):
    return session_idx * 1000000
```

- Session 1 的帧 0：全局节点 ID = 1000001
- Session 1 的锚点：全局节点 ID = 1000000
- Session 2 的帧 0：全局节点 ID = 2000001
- Session 2 的锚点：全局节点 ID = 2000000

这种编码方式将 session 索引编码为百万位，每个 session 最多容纳 999999 个节点，且所有图的节点 ID 全局唯一。

## BetweenFactorWithAnchoring：四变量因子

`raw/codes/lt-mapper/ltslam/include/ltslam/BetweenFactorWithAnchoring.h:L18-L127`

### 数学形式

这是一个 **4 变量**非线性因子 `NoiseModelFactor4<VALUE, VALUE, VALUE, VALUE>`，连接两个 session 的关键帧及其各自的 anchor：

$$\text{factor}(p_1, p_2, a_1, a_2) \quad \text{其中 } p_i: \text{帧位姿}, a_i: \text{anchor位姿}$$

### 误差计算

```cpp
Vector evaluateError(p1, p2, anchor_p1, anchor_p2, H1, H2, anchor_H1, anchor_H2):
    // Step 1: 局部→中心坐标系变换
    hx1 = Compose(anchor_p1, p1)    // hx1 = a1 ∘ p1，即 T_central^{i,k} = T_a^i · T_l^{i,k}
    hx2 = Compose(anchor_p2, p2)    // hx2 = a2 ∘ p2

    // Step 2: 计算两帧间的相对位姿
    hx = Between(hx1, hx2)          // hx = hx1 ⊖ hx2 = (hx1)⁻¹ · hx2

    // Step 3: 与测量值比较
    return Local(measured, hx)      // e = Log(measured⁻¹ · hx)
```

展开为数学表达式：

1. 将两帧各自变换到中心坐标系：
   - $T_g^{1} = T_a^{1} \cdot T_l^{1}$
   - $T_g^{2} = T_a^{2} \cdot T_l^{2}$

2. 计算中心坐标系下两帧间的相对位姿：
   - $T_{\text{relative}} = (T_g^{1})^{-1} \cdot T_g^{2}$

3. 与测量值比较（例如 ICP 计算得到的相对位姿 $T_{\text{measured}}$）：
   - $\mathbf{e} = \log\left(T_{\text{measured}}^{-1} \cdot T_{\text{relative}}\right)$

### 雅可比

GTSAM 的 `traits<T>::Compose` 和 `traits<T>::Between` 自动解析计算对四个变量的雅可比矩阵：

```cpp
T hx1 = traits<T>::Compose(anchor_p1, p1, anchor_H1, H1);  // H1 = ∂(a₁∘p₁)/∂p₁
T hx2 = traits<T>::Compose(anchor_p2, p2, anchor_H2, H2);  // H2 = ∂(a₂∘p₂)/∂p₂
T hx = traits<T>::Between(hx1, hx2, H1, H2);               // H1 = ∂(hx₁⊖hx₂)/∂hx₁
                                                             // H2 = ∂(hx₁⊖hx₂)/∂hx₂
```

关键：误差对 anchor 的雅可比通过链式法则自动传递，使 anchor 位姿也能在优化中更新。

```python
def initTrajectoryByAnchoring(sess):
    anchor_idx = genAnchorNodeIdx(sess.index_)
    if sess.is_base_session_:    # 中心会话
        gtSAMgraph.add(PriorFactor(anchor_idx, poseOrigin, priorNoise=1e-12))

```

**先验设计关键**：
- **中心会话 anchor**：强先验（噪声方差 ∼10⁻¹²），将其锚定在原点
- **查询会话 anchor**：弱先验（噪声方差 ∼π²/10⁸），允许优化器自由调整其变换
- 所有 anchor 初始化为单位位姿，优化后自动收敛到正确的坐标变换

### 添加 Session 内部图

```python
def addSessionToCentralGraph(sess):
    # 添加节点
    for node in sess.nodes_:
        global_idx = genGlobalNodeIdx(sess.index_, node.idx)
        if node.idx == 0:
            gtSAMgraph.add(PriorFactor(global_idx, curr_pose, priorNoise))
        initialEstimate.insert(global_idx, curr_pose)

    # 添加边
    for edge in sess.edges_:
        from_idx = genGlobalNodeIdx(sess.index_, edge.from)
        to_idx = genGlobalNodeIdx(sess.index_, edge.to)
        relative = edge.relative

        if abs(to_node - from_node) == 1:
            gtSAMgraph.add(BetweenFactor(from_idx, to_idx, relative, odomNoise))
        else:
            gtSAMgraph.add(BetweenFactor(from_idx, to_idx, relative, robustNoise))
```

注意：**内部边使用普通的 `BetweenFactor`**（仅 2 变量），不涉及 anchor。跨 session 的回环边才使用 `BetweenFactorWithAnchoring`（4 变量）。

### 添加跨 Session 回环

```python
def addSCloops():
    # 对等间隔采样的 SC 回环对
    for (target_node, source_node) in sampled_loop_pairs:
        # ICP 计算相对位姿（在局部坐标系中）
        relative_pose = doICPVirtualRelative(target_sess, source_sess, ...)

        # 添加四变量因子
        gtSAMgraph.add(BetweenFactorWithAnchoring<Pose3>(
            genGlobalNodeIdx(target_sess, target_node),   # key1: 目标帧
            genGlobalNodeIdx(source_sess, source_node),   # key2: 源帧
            genAnchorNodeIdx(target_sess),                 # anchor_key1
            genAnchorNodeIdx(source_sess),                 # anchor_key2
            relative_pose,                                 # 测量值
            robustNoise                                    # 鲁棒噪声
        ))
```

ICP 在**局部坐标系**中计算相对变换（使用 `loopFindNearKeyframesLocalCoord`），但因子约束的是**中心坐标系**中的关系，因此 anchor 位姿会受到约束。

| 噪声类型 | 方差（旋转 rad², 平移 m²） | 用途 |
|---------|---------------------------|------|
| priorNoise | (1e-12, 1e-12, 1e-12, 1e-12, 1e-12, 1e-12) | 中心会话 anchor 强先验 + 每 session 第一帧先验 |
| odomNoise | (1e-4, 1e-4, 1e-4, 1e-4, 1e-4, 1e-4) | 连续时间戳相邻帧间里程计边 |
| loopNoise | (1e-4, 1e-4, 1e-4, 1e-3, 1e-3, 1e-3) | 常规回环边 |
| largeNoise | (π², π², π², 1e8, 1e8, 1e8) | 查询会话 anchor 弱先验（允许自由移动） |
| robustNoise | Cauchy(1) wrapper × Diagonal | 跨 session 回环（含外点滤波） |

## 与 Atlas 多地图管理的对比

| 维度 | Atlas (ORB-SLAM3) | Anchor (lt-mapper) |
|------|-------------------|---------------------|
| **核心机制** | 地图合并：将关联帧焊接到目标地图，迁移全部元素 | 坐标变换：保持各 session 局部图独立，通过 anchor 对齐 |
| **坐标系** | 所有地图统一在同一个世界坐标系中 | 每个 session 保留局部坐标，通过 anchor 变换到中心坐标 |
| **合并方式** | 一次性批量迁移（关键帧 + 地图点） | 渐进式因子图优化（增量添加回环边） |
| **位姿优化** | 焊接 BA + EssentialGraph + 全局 BA | ISAM2 增量优化（所有因子联合） |
| **尺度处理** | Sim3（含尺度 s），处理单目尺度漂移 | SE3（无尺度），假设 LiDAR 里程计尺度已知 |
| **会话独立性** | 合并后原地图被标记为 Bad（废弃） | 每个 session 的局部图完整保留 |
| **适用场景** | 单目/双目/IMU 实时在线建图 | 多趟 LiDAR 离线建图拼接 |

## 伪代码：Anchor 多会话对齐完整流程

```python
# 伪代码
def multi_session_alignment(sessions):
    # 1. 初始化
    isam = ISAM2(relinearizeThreshold=0.1)
    graph = NonlinearFactorGraph()
    estimates = Values()
    priorNoise = DiagonalNoise(1e-12 * I_6)
    largeNoise = DiagonalNoise(π² * I_3, 1e8 * I_3)  # 旋转弱约束

    # 2. 为每个 session 添加 anchor 节点
    anchor_map = {}
    for sess in sessions:
        anchor_id = sess.index * 1_000_000
        anchor_map[sess.index] = anchor_id
        estimates.insert(anchor_id, Pose3.Identity())

        if sess.is_base:
            graph.add(PriorFactor(anchor_id, Pose3.Identity(), priorNoise))
        else:
            graph.add(PriorFactor(anchor_id, Pose3.Identity(), largeNoise))

    # 3. 添加各 session 内部图
    for sess in sessions:
        for node in sess.nodes:
            gid = sess.index * 1_000_000 + node.idx + 1
            estimates.insert(gid, node.initial_pose)
            if node.idx == 0:
                graph.add(PriorFactor(gid, node.initial_pose, priorNoise))
        for edge in sess.edges:
            from_gid = sess.index * 1_000_000 + edge.from + 1
            to_gid   = sess.index * 1_000_000 + edge.to   + 1
            graph.add(BetweenFactor(from_gid, to_gid, edge.relative, odomNoise))

    # 4. 初始优化（仅内部约束）
    isam.update(graph, estimates)

    # 5. 检测跨 session 回环（ScanContext + ICP）
    loop_pairs = detect_inter_session_loops(sessions)

    # 6. 添加跨 session 回环边（使用四变量因子）
    for (target_sess, target_node, source_sess, source_node) in loop_pairs:
        relative = icp_pair(target_sess, target_node, source_sess, source_node)
        if relative is valid:
            graph.add(BetweenFactorWithAnchoring(
                target_sess * 1e6 + target_node + 1,   # key1
                source_sess * 1e6 + source_node + 1,   # key2
                target_sess * 1e6,                      # anchor1
                source_sess * 1e6,                      # anchor2
                relative, robustNoise
            ))

    # 7. 最终优化
    isam.update(graph, estimates)
    isam.update()  # 多次 update 确保收敛

    # 8. 输出中心坐标系下的轨迹
    result_estimate = isam.calculateEstimate()
    for sess in sessions:
        anchor_pose = result_estimate.at(sess.anchor_id)
        for node in sess.nodes:
            local_pose = result_estimate.at(sess.local_node_ids[node.idx])
            global_pose = anchor_pose * local_pose
            output_trajectory(sess.index, node.idx, global_pose)
```

### 实现注意事项
- `kSessionStartIdxOffset` (1000000) 限制了每个 session 最多约 10⁶ 个节点，实际足够
- SC 回环的 ICP 在局部坐标系中进行，但因子在中心坐标系中定义——这是理解 `doICPVirtualRelative` 的关键
- Cauchy 鲁棒核对于跨 session 回环至关重要，因为 ScanContext 可能产生假阳性匹配
- ISAM2 的 `relinearizeSkip=1` 意味着每次更新都进行重线性化，适合批量添加因子后再统一优化的场景
- `pose_central = anchor_transform * pose_self_coord` 这里的乘法是 SE(3) 复合（`Pose3::operator*`），注意乘法顺序
