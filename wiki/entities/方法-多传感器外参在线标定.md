# 多传感器外参在线标定

> VINS-Fusion 中 IMU-相机外参 $( \mathbf{R}_{bc}, \mathbf{t}_{bc} )$ 的在线估计方法：从初始旋转手眼标定开始，随后在滑动窗口中将 6-DOF 外参作为优化变量进行持续精化。

## 问题动机

在 VIO 系统中，视觉特征的重投影误差依赖于相机到 IMU 的坐标变换链：

$$
\mathbf{p}_{\text{camera}} = \mathbf{R}_{bc}^{-1} \left[ \mathbf{R}_{b,w}^{-1} ( \mathbf{p}_w - \mathbf{t}_{b,w} ) - \mathbf{t}_{bc} \right]
$$

外参 $(\mathbf{R}_{bc}, \mathbf{t}_{bc})$ 的任何误差都会直接进入重投影残差，导致：
- **系统尺度估计偏差**
- **位姿估计漂移**
- **重投影误差虚高**（即使位姿和特征位置正确）
- **重力方向估计失真**

外参会因温度变化、机械应力、重新安装等原因漂移。在线标定能自动适应这些变化。

## VINS-Fusion 的三阶段标定策略

### 阶段 1：纯视觉相对旋转（solveRelativeR）

当 `ESTIMATE_EXTRINSIC == 2` 时，系统首先通过视觉特征匹配计算相邻帧的相对旋转：

1. 从连续两帧的 2D-2D 特征对应中计算本质矩阵 $\mathbf{E}$（OpenCV `findFundamentalMat`）
2. 分解 $\mathbf{E}$ 得到 4 组 $(\mathbf{R}, \mathbf{t})$ 解
3. 通过三角化重投影验证（`testTriangulation`），选择使得前方点最多的解
4. 输出相邻相机帧的相对旋转 $\mathbf{R}_{c_k}^{c_{k+1}}$

### 阶段 2：手眼标定（CalibrationExRotation）

利用视觉相对旋转和 IMU 预积分旋转，求解手眼标定方程：

$$
\mathbf{R}_{c_k}^{c_{k+1}} \cdot \mathbf{R}_{cb} = \mathbf{R}_{cb} \cdot \mathbf{R}_{b_k}^{b_{k+1}}
$$

其中 $\mathbf{R}_{b_k}^{b_{k+1}}$ 来自 IMU 预积分的 $\delta q$。

**求解方法**（四元数形式）：

对于每一对相邻帧 $i$，构造残差四元数：

$$
\mathbf{r}_i = \text{quat} \left( \mathbf{R}_{c,i} \right) - \text{quat} \left( \mathbf{R}_{cb} \cdot \mathbf{R}_{b,i} \cdot \mathbf{R}_{cb}^{-1} \right)
$$

将四元数乘法写成矩阵形式（通过四元数左乘/右乘矩阵 $\mathbf{L}$、$\mathbf{R}$），汇集多帧约束为超定线性系统 $\mathbf{A} \mathbf{x} \approx \mathbf{0}$（其中 $\mathbf{x}$ 是 $\mathbf{R}_{cb}$ 的四元数表示），用 SVD 求解：

```cpp
// raw/codes/VINS-Fusion/vins_estimator/src/initial/initial_ex_rotation.cpp:L29-L66
MatrixXd A(frame_count * 4, 4);
A.setZero();
for (int i = 1; i <= frame_count; i++) {
    Quaterniond r1(Rc[i]);    // 视觉相对旋转
    Quaterniond r2(Rc_g[i]);  // 融合了 ric 后的旋转链
    double huber = angular_distance > 5.0 ? 5.0 / angular_distance : 1.0;
    // 四元数左乘/右乘矩阵构造
    Matrix4d L, R;
    // L = 左乘矩阵(Rc[i]), R = 右乘矩阵(Rimu[i])
    A.block<4,4>((i-1)*4, 0) = huber * (L - R);
}
JacobiSVD<MatrixXd> svd(A, ComputeFullU | ComputeFullV);
Vector4d x = svd.matrixV().col(3);  // 最小奇异值对应向量
Quaterniond estimated_R(x);
ric = estimated_R.toRotationMatrix().inverse();
```

**收敛判定**：

```cpp
if (frame_count >= WINDOW_SIZE && ric_cov(1) > 0.25)
```

- 要求窗口填满（`WINDOW_SIZE` 帧，一般为 10 帧）
- 第二小奇异值 > 0.25，表明多组约束达成一致，旋转可观

### 阶段 3：滑动窗口精化

初始手眼标定成功后，系统切换到 `ESTIMATE_EXTRINSIC = 1`。在后续的滑动窗口优化中，外参 $(\mathbf{R}_{bc}, \mathbf{t}_{bc})$ 成为**可调整的参数块**。

**参数化**（与位姿相同，使用 Lie 代数局部参数化）：

```cpp
// raw/codes/VINS-Fusion/vins_estimator/src/estimator/estimator.cpp:L1025-L1038
for (int i = 0; i < NUM_OF_CAM; i++) {
    ceres::LocalParameterization *local_parameterization =
        new PoseLocalParameterization();
    problem.AddParameterBlock(para_Ex_Pose[i], SIZE_POSE,
                              local_parameterization);
    if ((ESTIMATE_EXTRINSIC && frame_count == WINDOW_SIZE
         && Vs[0].norm() > 0.2) || openExEstimation) {
        openExEstimation = 1;
    } else {
        problem.SetParameterBlockConstant(para_Ex_Pose[i]);
    }
}
```

**参数块布局**：
- `para_Ex_Pose[i]` 是 7 维数组：`[tx, ty, tz, qx, qy, qz, qw]`
- 使用 `PoseLocalParameterization` 将优化变量降到 6 维（3 维平移 + 3 维旋转 tangent space）
- 与位姿参数块完全相同的参数化方式

**可观测性守卫**：和 td 估计相同——仅在 `Vs[0].norm() > 0.2 m/s` 时激活。系统需要充分的角运动和线运动激励才能区分运动误差和外参误差。

## 数学公式

### 重投影中的外参

对于特征点在世界系中的坐标（由逆深度参数化），经过以下变换链投影到相机坐标系：

$$
\mathbf{p}_{\text{camera}} = \mathbf{R}_{bc}^{\mathsf{T}} \left[ \mathbf{R}_{b_j,w}^{\mathsf{T}} \left( \mathbf{R}_{b_i,w} \left( \mathbf{R}_{bc} \cdot \frac{\mathbf{p}_{\text{obs}}}{d} + \mathbf{t}_{bc} \right) + \mathbf{t}_{b_i,w} - \mathbf{t}_{b_j,w} \right) - \mathbf{t}_{bc} \right]
$$

$( \mathbf{R}_{bc}, \mathbf{t}_{bc} )$ 分别出现在：
1. **特征点从相机系到 IMU 系的转换**（$\mathbf{R}_{bc} \cdot \frac{\mathbf{p}_{\text{obs}}}{d} + \mathbf{t}_{bc}$）
2. **投影后从 IMU 系回到相机系的转换**（$\mathbf{R}_{bc}^{\mathsf{T}}$ 和 $-\mathbf{t}_{bc}$）

### Jacobian 对外参

外参 Jacobian 在 `ProjectionFactor::Evaluate` 和 `ProjectionTwoFrameOneCamFactor::Evaluate` 中实现，对旋转和平移分量分别求导：

**平移部分**：

$$
\frac{\partial \mathbf{e}}{\partial \mathbf{t}_{bc}} = \mathbf{J}_{\text{reduce}} \cdot \mathbf{R}_{bc}^{\mathsf{T}} \cdot \left( \mathbf{R}_{b_j}^{\mathsf{T}} \cdot \mathbf{R}_{b_i} - \mathbf{I} \right)
$$

**旋转部分**（使用 SO(3) 右扰动模型）：

$$
\begin{aligned}
\frac{\partial \mathbf{e}}{\partial \delta \boldsymbol{\theta}} = \mathbf{J}_{\text{reduce}} \cdot [\
& -\mathbf{R}_{bc}^{\mathsf{T}} \mathbf{R}_{b_j}^{\mathsf{T}} \mathbf{R}_{b_i} \mathbf{R}_{bc} \cdot \left[ \mathbf{p}_{\text{camera},i} \right]_\times \\
& + \left[ \mathbf{R}_{bc}^{\mathsf{T}} \mathbf{R}_{b_j}^{\mathsf{T}} \mathbf{R}_{b_i} \mathbf{R}_{bc} \cdot \mathbf{p}_{\text{camera},i} \right]_\times \\
& + \left[ \mathbf{R}_{bc}^{\mathsf{T}} \left( \mathbf{R}_{b_j}^{\mathsf{T}} ( \mathbf{R}_{b_i} \mathbf{t}_{bc} + \mathbf{t}_{b_i} - \mathbf{t}_{b_j} ) - \mathbf{t}_{bc} \right) \right]_\times \
]
\end{aligned}
$$

其中 $\mathbf{J}_{\text{reduce}}$ 是将 3D 误差映射到 2D 图像平面的约化矩阵（单位半球误差或归一化平面误差）。

## 伪代码：完整标定管线

```python
def calibrate_extrinsic_online():
    ESTIMATE_EXTRINSIC = 2  # 启动阶段：从零标定
    initial_ex_rotation = InitialEXRotation()
    ric = np.eye(3)
    tic = config.initial_tic  # 平移初值（通常设为 0 或粗测值）

    for each image_frame:
        # ---- 阶段 1+2：手眼标定 ----
        if ESTIMATE_EXTRINSIC == 2 and frame_count > 0:
            # 获取相邻帧的特征对应
            corres = feature_manager.get_corresponding(frame_count-1, frame_count)
            # IMU 预积分提供的相对旋转
            delta_q_imu = pre_integrations[frame_count].delta_q

            success = initial_ex_rotation.calibrate(corres, delta_q_imu)
            if success:
                ric = initial_ex_rotation.ric
                ESTIMATE_EXTRINSIC = 1  # 切换到精化模式

        # ---- 阶段 3：滑动窗口精化 ----
        if ESTIMATE_EXTRINSIC == 1 or ESTIMATE_EXTRINSIC == 2:
            problem = CeresProblem()

            # 外参参数块（使用流形局部参数化）
            for cam_id in range(NUM_OF_CAM):
                problem.add_parameter_block(
                    para_Ex_Pose[cam_id], dim=7,
                    local_param=PoseLocalParameterization()
                )
                # 可观测性守卫
                if frame_count == WINDOW_SIZE and velocity_norm > 0.2:
                    pass  # 保持为可优化
                else:
                    problem.set_parameter_block_constant(para_Ex_Pose[cam_id])

            # 添加 IMU 预积分约束
            for each frame_pair:
                problem.add_residual(IMUFactor(pre_integrations[j]))

            # 添加视觉重投影约束（外参在因子中使用）
            for each visible_feature:
                pts_cam_i = ric * (feature_3d / depth) + tic
                pts_w = R_i * pts_cam_i + t_i
                pts_cam_j = ric.T * (R_j.T * (pts_w - t_j) - tic)

                # 外参通过视觉因子参与优化
                problem.add_residual(ProjectionFactor(
                    pts_i, pts_j, pose_i, pose_j, para_Ex_Pose, depth
                ))

            problem.solve()

        # ---- 提取优化结果 ----
        if openExEstimation:  # 外参曾被激活优化
            ric = Quaternion(para_Ex_Pose[0][3:7]).to_rotation_matrix()
            tic = para_Ex_Pose[0][0:3]
```

## 观测性条件

外参（尤其是旋转分量）的在线可观性取决于**运动模式**：

| 运动类型 | 旋转可观 | 平移可观 | 说明 |
|----------|----------|----------|------|
| 纯平移 | ✓ (部分) | ✓ | 旋转 yaw 和 pitch 可观，roll 弱 |
| 纯旋转 | ✓ | ✗ | 平移不可观（无尺度信息） |
| 一般 6-DOF 运动 | ✓✓ | ✓✓ | 最佳激励 |
| 静止 | ✗ | ✗ | 所有外参退化 |

VINS-Fusion 使用 `Vs[0].norm() > 0.2` 作为简化的运动激励检测。更精确的在线置信度估计可使用 Fisher 信息矩阵的特征值。

## Agent 实现提示

### 适用场景
VIO 系统中 IMU-相机外参的在线校准（温度漂移、安装松动后的参数更新），以及无出厂标定时的启动自标定。

### 输入输出契约
- **输入**：IMU 测量（加速度计 + 陀螺仪），图像帧（单目/双目），特征跟踪结果，可选的初始外参
- **输出**：6-DOF 外参 $(\mathbf{R}_{bc}, \mathbf{t}_{bc})$，收敛置信度指标
- **前提**：系统处于运动激励中（`Vs.norm() > 0.2 m/s`），视觉特征匹配正常

### 实现骨架（伪代码）

```python
# 手眼标定：R_c * R_cb = R_cb * R_b
def hand_eye_calibration(Rc_list, Rb_list, angular_threshold=5.0):
    """四元数形式的 SVD 求解"""
    A = zeros(len(Rc_list) * 4, 4)
    for i, (Rc, Rb) in enumerate(zip(Rc_list, Rb_list)):
        q_c = quat(Rc)  # 视觉相对旋转
        q_b = quat(Rb)  # IMU 相对旋转
        ang_dist = angular_distance(q_c, quat(R_cb_guess * Rb * R_cb_guess.T))
        huber = 5.0 / ang_dist if ang_dist > 5.0 else 1.0

        L = quat_left_mult_matrix(q_c)
        R = quat_right_mult_matrix(q_b)
        A[i*4:(i+1)*4, :] = huber * (L - R)

    U, S, Vt = svd(A)
    q_cb = Vt[3, :]  # 最小奇异值对应向量
    return quat_to_R(q_cb).T  # 取逆得到 ric

# 外参投影因子（简化）
def projection_residual_ex(pts_i, pts_j, Pi, Qi, Pj, Qj, t_bc, q_bc, inv_depth):
    R_bc = q_bc.to_R()
    pts_cam_i = pts_i / inv_depth
    pts_imu_i = R_bc * pts_cam_i + t_bc
    pts_w = Qi * pts_imu_i + Pi
    pts_imu_j = Qj.inverse() * (pts_w - Pj)
    pts_cam_j = R_bc.T * (pts_imu_j - t_bc)
    e = (pts_cam_j / pts_cam_j.z()).head<2>() - pts_j.head<2>()

    # Jacobian w.r.t. 外参
    # dr/dt_bc = reduce * R_bc.T * (Rj.T * Ri - I)
    # dr/dtheta_bc = reduce * [-R_bc.T * Rj.T * Ri * R_bc * skew(pts_cam_i)
    #                           + skew(R_bc.T * Rj.T * Ri * R_bc * pts_cam_i)
    #                           + skew(R_bc.T * (Rj.T * (Ri * t_bc + Pi - Pj) - t_bc))]
    return e, dr_dt, dr_dtheta
```

### 实现注意事项
1. **外参方向约定**：VINS-Fusion 使用 $\mathbf{p}_{\text{imu}} = \mathbf{R}_{bc} \cdot \mathbf{p}_{\text{camera}} + \mathbf{t}_{bc}$（即 `ric`、`tic` 将相机系的点转换到 IMU 系）。检查你的系统是否使用相同的约定
2. **平移尺度**：在单目 VIO 中，$\mathbf{t}_{bc}$ 和全局尺度是耦合的——如果尺度未正确初始化，外参平移会吸收尺度误差
3. **平移初值**：手眼标定**只能估计旋转**（方程结构与平移无关），平移初值需要从物理测量或 Kalibr 离线标定获取
4. **收敛速度**：旋转通常比平移快收敛（5-10 次优化 vs 15-30 次），旋转收敛后平移的 Jacobian 条件数会改善
5. **与 td 的耦合**：$t_d$ 和外参平移存在耦合——两者都沿特征速度方向影响重投影位置。建议先固定 td 标定外参，或给两者添加弱先验正则化
6. **多相机处理**：双相机场景下，两个相机各自有独立的外参，视觉因子使用对应的外参。左目到 IMU、右目到 IMU 的外参同时优化
7. **外参先验**：如果出厂标定可靠，可在 Ceres 问题中对 `para_Ex_Pose` 添加正态先验残差，防止可观测性弱时外参漂移
