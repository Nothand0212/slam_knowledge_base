# 重力精细化（RefineGravity）

> VINS-Fusion 中通过切空间重参数化（tangent space reparameterization）对重力方向进行约束优化的方法。将重力从无约束 3D 向量降维为模长固定的 2-DoF 切平面参数，避免优化过程中重力模长漂移。

## 问题背景

在 `LinearAlignment()` 完成线性求解后，得到的重力向量 $\mathbf{g}$ 可能不满足模长约束 $|\mathbf{g}| = G \approx 9.81$。直接将 $\mathbf{g}$ 当作 3-DoF 自由向量进行非线性优化会引入以下问题：

1. **模长漂移**：优化器会把重力模长的误差分配到速度和尺度上，导致参数不一致。
2. **过参数化**：模长已知的物理约束未被充分利用，浪费 1 DoF 信息。
3. **收敛不稳定**：尺度 $s$ 和重力模长 $|\mathbf{g}|$ 在线性系统中高度耦合，自由优化容易发散。

## 切空间重参数化数学形式

重力向量的切空间展开：

$$\mathbf{g} = g_{\text{mag}} \cdot \bar{\mathbf{g}}_0 + w_1 \mathbf{b}_1 + w_2 \mathbf{b}_2$$

其中：

- $g_{\text{mag}} = G \approx 9.81$ 是重力模长（已知）
- $\bar{\mathbf{g}}_0 = \mathbf{g}_0 / |\mathbf{g}_0|$ 是当前重力估计的单位方向
- $\mathbf{b}_1, \mathbf{b}_2$ 是 $\bar{\mathbf{g}}_0$ 切平面的一组正交基
- $w_1, w_2$ 是重力方向在切平面上的修正量（待优化参数）

切空间基 $\mathbf{b}_1, \mathbf{b}_2$ 满足：

$$\mathbf{b}_1 \perp \bar{\mathbf{g}}_0, \quad \mathbf{b}_2 \perp \bar{\mathbf{g}}_0, \quad \mathbf{b}_1 \perp \mathbf{b}_2, \quad |\mathbf{b}_1| = |\mathbf{b}_2| = 1$$

重力方向修正后，重新归一化以保持模长：

$$\mathbf{g} \leftarrow \frac{\mathbf{g}_0 + w_1 \mathbf{b}_1 + w_2 \mathbf{b}_2}{|\mathbf{g}_0 + w_1 \mathbf{b}_1 + w_2 \mathbf{b}_2|} \cdot G$$

## 切空间基构造（TangentBasis）

源码位于 `raw/codes/VINS-Fusion/vins_estimator/src/initial/initial_aligment.cpp:L50–L63`。

```pseudo
function TangentBasis(g0):
    a = g0.normalized()               // 单位重力方向
    tmp = (0, 0, 1)
    if a == tmp: tmp = (1, 0, 0)      // 避免平行
    b = (tmp - a·(aᵀ·tmp)) / |...|    // Gram-Schmidt 正交化
    c = a × b                          // 叉积得到第二个正交基
    return [b | c] as 3×2 matrix
```

## RefineGravity 伪代码

源码位于 `raw/codes/VINS-Fusion/vins_estimator/src/initial/initial_aligment.cpp:L65–L133`。

```pseudo
function RefineGravity(all_image_frame, g, x):
    g0 = g.normalized() * G_magnitude      // 初始重力，模长已知方向未知

    for k = 0 to 3:                        // 4 次迭代足够收敛
        lxly = TangentBasis(g0)            // 在 g0 处构造切空间基

        for each adjacent frame pair (i, j):
            dt = pre_integration.sum_dt

            // 平移残差系数
            tmp_A_p.row(0..2, cols 0..2):  -dt * I₃                    // v_i
            tmp_A_p.row(0..2, cols 6..7):  Rᵢᵀ * ½dt² * I₃ * lxly    // w₁,w₂
            tmp_A_p.row(0..2, col  8):     Rᵢᵀ * (Tⱼ - Tᵢ) / 100.0   // s

            // 速度残差系数
            tmp_A_v.row(0..2, cols 0..2):  -I₃                         // v_i
            tmp_A_v.row(0..2, cols 3..5):  Rᵢᵀ * Rⱼ                   // v_j
            tmp_A_v.row(0..2, cols 6..7):  Rᵢᵀ * dt * I₃ * lxly      // w₁,w₂

            // 累积到正规方程 A*x = b
            A.add(tmp_Aᵀ * cov_inv * tmp_A)
            b.add(tmp_Aᵀ * cov_inv * tmp_b)

        // 求解
        A = A * 1000.0; b = b * 1000.0   // 数值稳定性缩放
        x = A.ldlt().solve(b)
        dg = x.segment<2>(n_state - 3)   // 提取 w₁, w₂

        // 更新重力方向（在切空间内修正）
        g0 = (g0 + lxly * dg).normalized() * G_magnitude

    g = g0
```

状态向量维度 `n_state = all_frame_count * 3 (速度) + 2 (w₁,w₂) + 1 (尺度)`。

## 与 LinearAlignment 的对比

| 特性 | LinearAlignment | RefineGravity |
|------|-----------------|---------------|
| 重力变量 | 3-DoF 向量 $\mathbf{g}$ | 2-DoF 切空间 $(w_1, w_2)$ |
| 尺度 $s$ | 优化变量 | 优化变量 |
| 速度 $\mathbf{v}_i$ | 优化变量 | 优化变量 |
| 重力模长约束 | 无（仅作为质量检查） | 内置（切空间保证） |
| 迭代次数 | 1 次（线性直接解） | 4 次（每次更新切空间基） |
| 残差项 | 位置 + 速度约束 | 位置 + 速度约束 |

## 为什么要在切空间内做

直接优化 $\mathbf{g} \in \mathbb{R}^3$ 时，$\mathbf{g}$ 与尺度 $s$ 的耦合使 Jacobian 病态：

$$\alpha_{b_i b_j} \approx \mathbf{R}_w^{b_i} (\mathbf{p}_w^{b_j} - \mathbf{p}_w^{b_i} - \mathbf{v}_i^w \Delta t + \frac{1}{2} \mathbf{g}^w \Delta t^2)$$

其中 $\mathbf{g}^w$ 和 $s$（隐含在 $\mathbf{p}_w^{b_j} - \mathbf{p}_w^{b_i}$ 中）通过 $\Delta t^2$ 耦合。切空间参数化将 $\mathbf{g}$ 的方向自由度分离出来，尺度负责吸收模长误差，避免了两者互相"解释"对方残差的病态。

## 工程细节

1. **数值缩放**：`A * 1000.0` 是为了提高 LDLT 分解的数值稳定性，VINS-Fusion 中尺度 $s$ 除以 100 放入状态变量，防止大数值导致矩阵条件数恶化（`raw/codes/VINS-Fusion/vins_estimator/src/initial/initial_aligment.cpp:L99`）。
2. **固定迭代 4 次**：由于每次迭代后切空间基 $\mathbf{b}_1, \mathbf{b}_2$ 都会在 $\mathbf{g}_0$ 处重新计算，是非线性 Gauss-Newton 风格更新。实际中 4 次足以收敛到 $< 0.1^\circ$ 方向误差。
3. **cov_inv 设置为单位阵**：当前代码未使用 IMU 噪声协方差的逆加权（`initial_aligment.cpp:L111`, `cov_inv.setIdentity()`），意味着预积分残差被等权重对待。
