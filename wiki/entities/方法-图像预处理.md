# 图像预处理

> 图像预处理是视觉 SLAM/VIO 中把原始图像转成可观测量的前置环节；它既可能是几何坐标变换，也可能是特征跟踪增强、光度残差建模或神经网络输入规范。

## 简介

在视觉 SLAM/VIO 中，图像预处理不是单独的“图像增强”模块，而是和观测模型绑定。KLT/ORB 类间接法主要需要稳定特征；直接法需要可解释的光度一致性；学习式/稠密方法需要满足训练时的输入分布；LIVO/RGB map 还会把曝光、时间同步和颜色建图耦合在一起。

## 预处理类型

| 类型 | 典型操作 | 代表系统 | 主要影响 |
|------|----------|----------|----------|
| 几何变换 | resize、downsample、crop、rectify、undistort、remap | OpenVINS、ORB-SLAM3、Kimera-VIO、DROID-SLAM、NICE-SLAM、R3LIVE | 改变像素坐标和相机内参，影响 bearing、reprojection 和 ray direction |
| 特征增强 | CLAHE、histogram equalization、灰度化、mask/grid、金字塔 | OpenVINS、LVI-SAM、VINS-Fusion、MSCKF、Kimera-VIO | 改变角点数量、KLT 成功率、RANSAC inlier ratio 和 track lifetime |
| 光度建模 | response、vignette、exposure、affine brightness、inverse exposure | DSO、DM-VIO、ROVIO、SVO、FAST-LIVO2、MonoGS | 直接进入光度残差或优化状态，影响残差物理意义 |
| 网络输入规范 | RGB/BGR 顺序、`/255`、mean/std、resize/crop 后内参更新 | DROID-SLAM、NICE-SLAM、MonoGS | 决定推理输入是否匹配训练分布和可微渲染几何 |

## 与观测模型的关系

### KLT/角点前端

CLAHE、均衡化和金字塔通常服务于 [[方法-视觉特征跟踪]]。它们的目标是增加可跟踪角点、扩大运动捕获范围，并通过 mask/grid 保持空间分布。评估时应看 tracked count、new feature count、RANSAC inlier ratio、track lifetime，而不是只看 ATE。

### ORB/描述子前端

ORB/描述子系统更依赖尺度一致性和 stereo rectification。resize 或 remap 后必须保证相机矩阵和 keypoint 坐标一致，否则 ORB pyramid、双目 disparity、BoW 匹配和重投影残差都会受影响。

### 直接法/半直接法

直接法的“预处理”是 [[概念-直接法光度误差]] 的一部分。response/vignette/exposure/affine brightness 应被视为残差模型或状态变量；任意 CLAHE 会改变局部亮度映射，使同一空间点的灰度差不再只对应曝光或仿射亮度变化。

### 学习式/稠密方法

DROID-SLAM、NICE-SLAM 和 MonoGS 更像是输入契约问题。颜色顺序、归一化、分辨率和内参更新需要与训练或渲染配置一致；传统增强不一定提升效果，反而可能造成训练-推理分布偏移。

## 工程检查清单

1. 输入图像是 RGB、BGR、GRAY 还是 float tensor。
2. 是否做了 resize、downsample、crop、rectify、undistort。
3. 内参 `fx/fy/cx/cy` 和图像尺寸是否同步更新。
4. 点坐标处于 distorted pixel、rectified pixel、normalized plane 还是 bearing vector。
5. 预处理是否改变了亮度物理含义。
6. 阈值单位是 pixel、normalized coordinate、pyramid level pixel 还是 intensity。
7. 后端残差是 reprojection、bearing、photometric patch、dense BA 还是 ray depth/color。

## Agent 实现提示

### 适用场景

当 agent 需要为 VIO/SLAM 前端新增或排查图像预处理时，先按观测模型选择分支：特征法优先考虑灰度化、CLAHE/均衡化、mask/grid、KLT 金字塔；ORB/双目系统优先保证 rectification、resize 和金字塔尺度一致；直接法优先保留光度物理意义；学习式系统优先匹配训练输入契约。

### 输入输出契约

- 输入：原始图像、时间戳、camera intrinsics/distortion、mask、曝光时间/增益、预处理配置。
- 输出：处理后的图像、同步更新后的 intrinsics/分辨率/mask、颜色空间标记、可选曝光元数据。
- 坐标/单位：resize/crop/remap 后的点是 processed pixel；稀疏点去畸变后通常是 normalized plane；网络输入常是 RGB float tensor，强度范围由 `/255` 和 mean/std 决定。

### 实现骨架（伪代码）

```pseudo
function preprocess_frame(frame, camera, config):
    image = frame.image
    mask = frame.mask
    K = camera.intrinsics

    if config.rectify_or_undistort:
        image = remap(image, camera.rectify_map)
        mask = remap(mask, camera.rectify_map)

    if config.resize_or_crop:
        image, mask, K = resize_crop_and_update_intrinsics(image, mask, K, config)

    if config.frontend == "klt" or config.frontend == "corner":
        gray = ensure_gray(image, config.color_order)
        gray = optional_histogram_or_clahe(gray, config.histogram_method)
        pyramid = build_optical_flow_pyramid(gray, config.win_size, config.pyr_levels)
        return PreprocessedFrame(gray, pyramid, mask, K, frame.timestamp)

    if config.frontend == "direct":
        irradiance = optional_response_vignette_exposure(image, frame.exposure, camera.photometric_calib)
        return PhotometricFrame(irradiance, K, frame.exposure, frame.timestamp)

    if config.frontend == "learned":
        tensor = normalize_for_network(image, config.color_order, config.mean, config.std)
        return NetworkFrame(tensor, K, frame.timestamp)
```

```python
# resize image
sx = self.image_size[1] / image.shape[2]
sy = self.image_size[0] / image.shape[1]

image = F.interpolate(image[None], self.image_size, mode='bilinear', align_corners=False)[0]

fx, fy, cx, cy = intrinsic.unbind(dim=0)
fx, cx = sx * fx, sx * cx
fy, cy = sy * fy, sy * cy

# crop image
if self.crop_size[0] > 0:
    cy = cy - self.crop_size[0]
    image = image[:,self.crop_size[0]:-self.crop_size[0],:]

if self.crop_size[1] > 0:
    cx = cx - self.crop_size[1]
    image = image[:,:,self.crop_size[1]:-self.crop_size[1]]

intrinsic = torch.stack([fx, fy, cx, cy])
```

### 实现注意事项

- `cv::pyrDown`、`cv::resize`、crop、rectify 和 remap 都会改变像素坐标解释；必须同时更新 mask、分辨率和 `fx/fy/cx/cy`。
- CLAHE/均衡化适合 KLT/角点前端的对比度增强，但不要默认放进直接法光度残差前，因为它会破坏亮度一致性。
- 学习式系统要优先复刻训练时的输入契约，例如 BGR/RGB 顺序、`/255`、mean/std、分辨率和 crop 策略。
