# 回环验证方法族

> 回环检测中几何验证阶段的统一知识页。涵盖四阶段验证（FAST-LIO-SAM-SC-QN）到五重验证（4DRadarSLAM）的两条工程验证管线，并抽象出广义 N 阶段验证框架。

## 概述

回环检测的完整流程分为两个阶段：**候选召回**（地点识别）和**几何验证**（候选确认）。候选召回用描述子（ScanContext、DBoW2、ISC）在海量历史帧中快速检索"可能为回环"的候选帧，但描述子匹配存在根本局限性：

1. **感知歧义**：对称结构、重复纹理可能导致高相似度但非同一地点。
2. **外观变化**：光照、季节、动态物体改变观测但不改变位置。
3. **方向模糊**：描述子旋转不变性可能与真实位姿的 yaw 偏移不完全对应。
4. **几何空洞**：描述子只编码观测的统计特征，不编码 "两帧点云能否对齐" 的几何约束。

因此 **几何验证不可省略** —— 必须用点云配准和一致性检查确认 "候选帧不仅是相似地点，而且是同一地点且相对位姿可用"。本族描述两条经过工程验证的递进式几何验证管线，以及从它们中抽象出的统一 N 阶段框架。

### 验证层级的根本逻辑

每一层验证解决一类不同的失败模式。层数增加不是冗余，而是对失败模式空间的完备覆盖：

```
描述子匹配 → 【层0】距离/时间先验过滤 → 【层1】粗几何配准 → 【层2】精细配准 → 【层3】单边约束一致性 → 【层4】多边约束交叉验证
    ↓              ↓                    ↓               ↓                ↓                  ↓
 召回候选       排除明显不可能        排除初值不适配     排除局部最优点    排除与里程计矛盾     排除与已有回环矛盾
```

**关键洞察**：单靠配准 score/收敛与否做几何验证是危险的——即使在错误对应上 NDT/ICP 也可能在局部盆地"收敛"。一致性检查（odometry check、pairwise check）通过外部约束兜底，覆盖配准本身的验证盲区。

## 对比表

### 四阶段 vs 五重验证

| 维度 | [[方法-四阶段回环验证]] | [[方法-五重回环几何验证]] |
|------|------------------------|--------------------------|
| **阶段数** | 4 | 5 |
| **适用传感器** | 机械旋转 LiDAR（稠密、360°） | 4D 成像毫米波雷达（稀疏、FOV 有限） |
| **代表框架** | FAST-LIO-SAM-SC-QN | 4DRadarSLAM |
| **第 0 层（预筛选）** | 无专门的预筛选层 | 距离/累计距离/高度差/yaw 差/椭圆模型 |
| **第 1 层（描述子）** | ScanContext 列对齐 + 余弦距离 | Intensity Scan Context (ISC) Ring Key + 列对齐 |
| **第 2 层（粗过滤）** | 欧氏距离 < 35m（用当前位姿） | ICP fitness score < 阈值（如 6.0） |
| **第 3 层（粗配准）** | Quatro FPFH + GNC（对初值不敏感） | Odometry Check：单边 T_err 与里程计对比 |
| **第 4 层（精配准）** | Nano-GICP FastGICP 分布匹配 + score | Pairwise Consistency：新回环与已有回环交叉验证 |
| **精配准深度** | 深（GICP 马氏距离 + 协方差建模） | 浅（标准 ICP，传感器稀疏限制了精配准收益） |
| **一致性检查** | 无专门的一致性子阶段 | 有（odometry check + pairwise check，两层） |
| **回环约束** | 配准 score 作为信息矩阵 | Huber 鲁棒核（size=1.0） |
| **设计哲学** | 配准驱动——依靠配准质量和信息矩阵 | 一致性驱动——依靠多边验证排除误匹配 |

### 为什么 4 vs 5？

两者的阶段数差异不是设计质量的差异，而是 **传感器特性决定的**：

| 因素 | LiDAR 特点 | 雷达特点 | 对验证策略的影响 |
|------|-----------|---------|---------------|
| 点密度 | 高（数万点/帧） | 低（数千点/帧） | 雷达配准更易失败，需要更多一致性兜底 |
| 噪声模型 | 各向同性，~2cm | 强各向异性，距离方向 ~10cm | 雷达配准 score 的判别力弱 |
| FOV | 360° 全覆盖 | 有限 FOV（~120°） | 雷达面临的对称歧义更少，但欠约束方向更多 |
| 回环代价 | 加入错误回环影响全局精度 | 错误回环可能完全破坏地图 | 雷达需要更严格的拒绝策略 |

LiDAR 的稠密点云让 GICP 精配准本身就具有高判别力（正确回环 fitness ~0.01 vs 错误回环 fitness ~0.5），所以 FAST-LIO-SAM 可以把多数验证职责交给配准本身。雷达的稀疏性让 ICP fitness 成为弱判别器（正确 1.0 vs 错误 3.0，阈值 6.0 只能过滤极端错误），所以 4DRadarSLAM 增加了 pre-filter 和 pairwise consistency 来补足。

## 统一视角

### 广义 N 阶段回环验证框架

从两个具体管线中抽象出通用的回归验证框架。给定传感器类型和场景，选择 $N$ 个阶段的组合和配置：

```text
function GeneralizedLoopVerification(candidate, curKF, sensor_profile, config):
    // == Stage 0: 先验过滤（可选）==
    if not config.skip_prefilter:
        if euclidean_distance(curKF, candidate) > config.max_dist: reject
        if accumulated_travel < config.min_accum_dist: reject
        if abs(baro_altitude_diff) > config.max_baro_diff: reject
        if abs(yaw_diff) > config.max_yaw_diff: reject
    
    // == Stage 1: 描述子度量（必须）==
    desc_sim = descriptor_distance(curSC, candidate.SC)
    if desc_sim < config.desc_sim_th: reject
    
    // == Stage 2: 快速几何门槛（可选）==
    // 粗配准前做一次廉价距离或 FOV 检查
    if not config.skip_geo_precheck:
        if coarse_euclidean(curKF, candidate, T_odom) > config.coarse_dist_th: reject
    
    // == Stage 3: 粗几何配准（推荐）==
    // 选择：Quatro(GNC) / NDT 低分辨率 / ISS+FPFH RANSAC
    T_coarse = coarse_align(curCloud, candidate.cloud, init_from_odom)
    if alignment_score(T_coarse) < config.coarse_score_th: reject
    
    // == Stage 4: 精几何配准（推荐）==
    // 选择：GICP / VGICP / Nano-GICP / 高分辨率 NDT
    T_fine, fitness, info = fine_align(curCloud, candidate.cloud, T_coarse)
    if fitness > config.fine_fitness_th: reject
    
    // == Stage 5: 单边约束一致性（推荐）==
    // 对稀疏传感器必须，对稠密 LiDAR 可选
    if config.enable_odom_check:
        T_err = T_fine * T_odom  // 回环变换与里程计的差异
        trans_err_per_frame = t_err.norm() / num_intermediate_frames
        if trans_err_per_frame > config.odom_trans_th: reject
        if rot_err_per_frame > config.odom_rot_th: reject
    
    // == Stage 6: 多边约束一致性（可选）==
    // 高安全性要求的场景
    if config.enable_pairwise_check:
        for existing_loop in loop_graph:
            if not pairwise_consistent(T_fine, existing_loop, config): reject
    
    // == 信息矩阵构建 ==
    if sensor_profile == DENSE_LIDAR:
        info = fitness_to_information(fitness)
    elif sensor_profile == SPARSE_RADAR:
        info = fixed_information * huber_weight
    
    return (T_fine, info, valid)
```

### 阶段选择决策表

| 场景 | 推荐阶段组合 | 理由 |
|------|------------|------|
| 室内 LiDAR，结构丰富 | 1 + 3 + 4 | 结构好，配准判别力强，一致性可省略 |
| 户外 LiDAR，混合地形 | 0 + 1 + 3 + 4 | 加预筛选排除时间过近的候选 |
| 4D 雷达，稀疏点云 | 0 + 1 + 2 + 3 + 5 + 6 | 全阶段覆盖，一致性格外重要 |
| 多传感器融合 | 0 + 1 + 3 + 4 + 5 | LiDAR 配准 + IMU/里程计交叉验证 |

### 通用坐标系与单位约定

| 量 | 坐标系 | 单位 | 说明 |
|----|--------|------|------|
| 欧氏距离预筛选 | 世界坐标系 xy 平面 | m | 使用当前 LIO/里程计位姿估计，而非全局优化后位姿 |
| 高度差/气压计差 | 世界坐标系 z / 气压高度 | m | 对平面运动的机器人更有效，3D 运动（无人机）禁用 |
| 偏航角差 | 世界坐标系 yaw | rad 或 degree | 注意 $\pm\pi$ 环绕；使用角度差最小化函数 |
| 描述子距离 | 描述子自身度量空间 | 取决于描述子 | ScanContext 用余弦距离；DBoW2 用分数 |
| 配准 fitness | 点云空间（公共坐标系） | 无单位 | fitness = 内点均方残差（受配准方法和分辨率影响） |
| odom check 平移 | 每个中间帧的局部增量 | m/frame | 除以帧数归一化为每帧漂移率 |
| pairwise check 平移 | 全局坐标系 | m | 绝对偏差而非每帧归一化 |

## Agent 实现提示

### 适用场景

- 为 LiDAR SLAM 或 4D 雷达 SLAM 实现回环检测的几何验证模块。
- 需要根据传感器类型和场景特性选择验证阶段组合。
- 需要在回环可靠性（recall）和安全性（precision）之间做工程权衡。
- 不适用于纯视觉回环（视觉回环的几何验证使用 PnP RANSAC、Sim3/SE3 重投影优化等不同机制，参见 [[方法-视觉回环检测管线]]）。

### 输入输出契约

- **输入**：
  - 当前关键帧：点云（已去畸变、降采样）、当前位姿估计（来自里程计/LIO）、描述子（ScanContext/ISC）
  - 候选关键帧列表：每个候选含点云（或可构建子地图）、位姿、描述子
  - 配置参数：各阶段阈值、配准方法选择、一致性检查开关
  - 历史回环边（pairwise check 需要）
- **输出**：
  - 通过验证的回环约束：相对位姿 $T_{ij}$、信息矩阵/协方差、诊断量（各阶段得分）
  - 被拒绝的候选及拒绝原因（用于调试和参数调优）
- **前置条件**：描述子库已建好索引；候选帧的点云可通过关键帧 ID 访问；里程计位姿可靠
- **坐标系**：所有位姿均为世界坐标系下的 SE(3)；$T_{ij}$ 为 j（候选帧）到 i（当前帧）的变换

### 实现骨架（伪代码）

```text
function LoopVerificationPipeline(cur_kf, candidate_kfs, config):
    verified_loops = []
    
    for (cand, kf_id, init_T_odom) in candidate_kfs:
        // == Stage 0: 先验过滤 ==
        if config.prefilter.enabled:
            if not check_distance_interval(cur_kf, cand): reject("dist_interval")
            if not check_accum_travel(cur_kf): reject("accum_travel")
            if not check_attitude_diff(cur_kf, cand): reject("attitude")
        
        // == Stage 1: 描述子距离（通常已在召回阶段完成，此处可选重复校验）==
        desc_dist = descriptor_distance(cur_kf.desc, cand.desc)
        if desc_dist > config.desc_threshold: reject("descriptor")
        
        // == Stage 2: 快速几何门槛 ==
        if config.geo_precheck.enabled:
            xy_dist = (cur_kf.pose.translation() - cand.pose.translation()).norm()
            if xy_dist > config.geo_precheck.max_xy_dist: reject("geo_xy")

        // == Stage 3: 粗几何配准 ==
        T_coarse = coarse_registration(cur_kf.cloud, cand.cloud,
                                        init_T_odom, config.coarse_method)
        if T_coarse is None: reject("coarse_failed")
        coarse_score = evaluate_alignment(T_coarse, cur_kf.cloud, cand.cloud)
        if coarse_score > config.coarse_score_th: reject("coarse_score")
        
        // == Stage 4: 精几何配准 ==
        T_fine, fine_score, info = fine_registration(cur_kf.cloud, cand.cloud,
                                                      T_coarse, config.fine_method)
        if fine_score > config.fine_fitness_th: reject("fine_fitness")
        
        // == Stage 5: 单边 Odometry Check ==
        if config.odom_check.enabled:
            T_err = T_fine * cand.T_odom_to_cur.inverse()
            trans_per_frame = T_err.translation().norm() / cand.num_frames
            rot_per_frame = angle(T_err.rotation()) / cand.num_frames
            if trans_per_frame > config.odom_check.trans_th: reject("odom_trans")
            if rot_per_frame > config.odom_check.rot_th: reject("odom_rot")
        
        // == Stage 6: 多边 Pairwise Check ==
        if config.pairwise_check.enabled and not loop_graph.empty():
            for existing in loop_graph:
                if not check_pairwise_consistency(T_fine, existing,
                                                   config.pairwise_check): reject("pairwise")
        
        // == 构建约束 ==
        verified_loops.append({
            T_ij: T_fine,
            information: construct_information(fine_score, config.sensor_profile),
            diagnosis: {stage_scores...}
        })
    
    return verified_loops
```

### 实现注意事项

- **配准 score 不是总可靠**：错误对应上的 NDT probability 和 ICP fitness 可能在局部盆地给出合格分数。稠密 LiDAR 可更多倚重配准 score（判别力强），稀疏雷达必须额外依赖一致性检查（odometry check + pairwise check）。
- **阈值参数是传感器/场景特定的**：将关键阈值（fitness、odom_trans_th、pairwise_trans_th）做成可配置参数或 YAML 节点，方便不同部署环境调优。不要在代码中硬编码。
- **信息矩阵的构建策略因传感器而异**：LiDAR 回环可从 GICP Hessian 中提取 6x6 信息矩阵（协方差求逆）；雷达回环因配准质量不稳定，常使用固定信息矩阵 + Huber 鲁棒核（size ~1.0），避免错误回环压倒里程计约束。
- **Pairwise check 的计算量可控**：只检查已有回环边中与新回环节点几何相关的边（通过节点 ID 交集判断），而非遍历全图所有边。
- **Odom check 的帧间归一化很重要**：直接用绝对平移误差比较不公平——两帧间隔 100 帧和间隔 5 帧的里程计漂移量级不同。必须将平移/旋转误差除以中间帧数。
- **预筛选的顺序有性能影响**：先做最便宜的距离/时间检查，再做较贵的描述子距离计算，最后执行最昂贵的点云配准。每层都应在进入下一层前过滤掉尽可能多的候选。
- **拒绝原因日志对调试至关重要**：记录每个被拒绝候选的失败阶段和度量值，方便事后分析是阈值过严还是传感器退化了。
