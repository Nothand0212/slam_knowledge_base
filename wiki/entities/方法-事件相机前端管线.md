# 事件相机前端管线（ESVO）

> ESVO（Event-based Stereo Visual Odometry）是首个完整的纯事件双目视觉里程计系统。前端将异步事件流转换为 Time Surface 表示，在立体图像对上执行事件级匹配（Time Surface 上的 ZNCC block matching），并通过非线性优化精化深度估计。核心创新在于用事件流的极高时间分辨率替代帧图像，实现了 μs 级时间精度的连续视觉里程计。

## 系统架构

ESVO 由两个解耦的并行系统组成：

```
事件流 (左/右)
    │
    ├─ Time Surface 生成（指数衰减 → 2D 图像）
    │
    ├─ Mapping 线程 ─────────────────────────────────
    │   ├─ 事件切片（按时间窗口分组）
    │   ├─ Block Matching（ZNCC 在 TS 上的双目匹配）
    │   ├─ 非线性深度优化（Gauss-Newton / LM）
    │   ├─ 深度融合（多帧融合 + 滤波）
    │   └─ 发布 DepthMap + 点云 → Tracking
    │
    └─ Tracking 线程 ────────────────────────────────
        ├─ 接收 DepthMap（参考帧）
        ├─ 在当前 TS 上做直接法位姿追踪
        ├─ 非线性优化（LM 最小化 TS 残差）
        └─ 发布 6-DoF 位姿
```

**源码锚点**: Mapping: `raw/codes/ESVO/esvo_core/include/esvo_core/esvo_MVStereo.h:L51-L242`；Tracking: `raw/codes/ESVO/esvo_core/include/esvo_core/esvo_Tracking.h:L43-L134`

## 事件数据表示

### 事件流 → Time Surface

原始事件数据格式：

$$
e_k = (x_k, y_k, t_k, p_k)
$$

其中 $(x_k, y_k)$ 为像素坐标，$t_k$ 为 μs 级时间戳，$p_k \in \{-1, +1\}$ 为极性（亮度下降/上升）。

Time Surface（TS）将事件流转换为 2D 连续值图像（详见 [[方法-Time-Surface]]）：

$$
TS(x, y) = p_{\text{latest}} \cdot \exp\big(-\frac{T_{\text{ref}} - t_{\text{latest}}}{\tau}\big)
$$

其中 $\tau$ 为衰减常数（典型值 30ms），$T_{\text{ref}}$ 为参考时间。

**源码锚点**: `raw/codes/ESVO/esvo_core/include/esvo_core/container/TimeSurfaceObservation.h:L27-L157`

在 ESVO 实现中，TS 还需计算 Sobel 梯度用于后续优化：

```cpp
cv::Sobel(left->image, cv_dTS_du_left, CV_64F, 1, 0);
cv::Sobel(left->image, cv_dTS_dv_left, CV_64F, 0, 1);
```

高斯模糊预处理（kernelSize=5）用于降低事件噪声的影响：
```cpp
cv::GaussianBlur(cvImagePtr_left_->image, mat_left_, cv::Size(5, 5), 0.0);
```

### Negative Time Surface

为了补充 OFF 事件（亮度下降）的信息，ESVO 定义 Negative Time Surface：

$$
TS_{\text{neg}}(x, y) = 255 - TS_{\text{blurred}}(x, y)
$$

**源码锚点**: `raw/codes/ESVO/esvo_core/include/esvo_core/container/TimeSurfaceObservation.h:L118-L134`

## 立体匹配：Event Block Matching

### 匹配流程

ESVO 的 mapping 核心是在左右 Time Surface 上做 ZNCC（Zero-mean Normalized Cross-Correlation）块匹配：

```
for each event at (x_left, y_left):
    1. 提取左 TS 上以事件为中心的 patch (patch_size_X × patch_size_Y)
    2. 检查 patch 有效性：
       - 不在图像边界（距边缘 ≥ 1 像素，为插值留空间）
       - 非退化 patch（>95% 像素 TS 值 < 1 则为噪声区，跳过）
    3. 沿极线在右 TS 上搜索（步长 step_）：
       - 对每个候选视差 disp ∈ [min_disp, max_disp]，提取右 TS patch
       - 计算 ZNCC cost = 0.5*(1 - corr_normalized)
    4. 粗搜索：step_ 步长找全局最小 ZNCC
    5. 精搜索：在粗搜索结果 ±(step_-1) 范围内以步长 1 精细定位
    6. cost < ZNCC_Threshold → 生成 EventMatchPair
```

**源码锚点**: `raw/codes/ESVO/esvo_core/src/core/EventBM.cpp:L80-L168` (match_an_event)，`L170-L226` (epipolarSearching)

### ZNCC 代价函数

```cpp
double zncc_cost(Eigen::MatrixXd &patch_left, Eigen::MatrixXd &patch_right, bool normalized) {
    Eigen::MatrixXd patch_left_normalized, patch_right_normalized;
    tools::normalizePatch(patch_left, patch_left_normalized);
    tools::normalizePatch(patch_right, patch_right_normalized);
    double cost = 0.5 * (1 - (patch_left_normalized.array()
                             * patch_right_normalized.array()).sum()
                             / (patch_left.rows() * patch_left.cols()));
    return cost;
}
```

**源码锚点**: `raw/codes/ESVO/esvo_core/src/core/EventBM.cpp:L317-L333`

### 稀疏深度计算

匹配成功后，通过视差直接计算深度：

$$
d_{\text{depth}} = \frac{B \cdot f}{\text{disparity}}
$$

其中 $B$ 为基线长度，$f$ 为焦距（取内参矩阵 $P(0,0)$）。逆深度 $1/d$ 作为后续优化的初始值。

```cpp
double disparity = x1(0) - bestMatch(0);  // 水平配置
double depth = camSysPtr_->baseline_ * camSysPtr_->cam_left_ptr_->P_(0,0) / disparity;
emPair.invDepth_ = 1.0 / depth;
```

**源码锚点**: `raw/codes/ESVO/esvo_core/src/core/EventBM.cpp:L141-L168`

## 非线性深度优化

### 问题形式化

对每个 EventMatchPair，定义点 $P$ 在参考帧（左相机坐标系）中的 3D 位置：

$$
P = \begin{bmatrix} \frac{x - c_x}{f_x} \\ \frac{y - c_y}{f_y} \\ 1 \end{bmatrix} \cdot \frac{1}{\rho}
$$

其中 $\rho$ 为待优化的逆深度。将点映射到目标帧（右相机）：

$$
\begin{bmatrix} u' \\ v' \end{bmatrix} = \pi_{\text{right}}\big(T_{\text{right}\leftarrow\text{left}} \cdot P\big)
$$

残差定义为采样点与参考 patch 的归一化光度差：

$$
r = \mathcal{I}_{\text{left}}(u, v) - \mathcal{I}_{\text{right}}(u', v')
$$

其中 $\mathcal{I}$ 为 Time Surface 强度（双线性插值采样）。

**源码锚点**: `raw/codes/ESVO/esvo_core/include/esvo_core/optimization/OptimizationFunctor.h:L16-L63` (Eigen 非线性优化 functor 基类)，`raw/codes/ESVO/esvo_core/include/esvo_core/core/DepthProblemSolver.h` (具体求解器)

### 优化策略

`DepthProblemSolver` 使用 Eigen 的 `LevenbergMarquardt` 对每个匹配点的逆深度执行 1-DoF 优化。选项包括：

- **NUMERICAL**：数值雅可比（Eigen 自动差分）
- **ANALYTICAL**：解析雅可比（手动链式法则）
- **TDistribution**：T 分布鲁棒范数替代 L2，对 outliers 更鲁棒

优化 10 次迭代（`ITERATION_OPTIMIZATION = 10`）即可收敛。

**源码锚点**: `raw/codes/ESVO/esvo_core/src/esvo_Mapping.cpp:L37-L45` (DepthProblemConfig 构造)

## 深度滤波与融合

### 帧内正则化

对 DepthMap 做局部邻域正则化（median / bilateral filter）：

- 半径 `RegularizationRadius = 5`
- 最小邻居数 `RegularizationMinNeighbours = 8`
- 移除孤立的异常深度点

**源码锚点**: `raw/codes/ESVO/esvo_core/src/core/DepthRegularization.cpp`

### 帧间融合

`DepthFusion` 将多帧的深度估计融合到统一的 DepthMap 中：

- 策略 `CONST_FRAMES`：保留最近 N 帧的深度点并做加权平均
- 每个深度点维护：`invDepth`, `variance`, `age`, `cost`
- 滤波阈值：`maxNumFusionFrames=10`, `residual_vis_threshold=15`, `stdVar_vis_threshold=0.005`

**源码锚点**: `raw/codes/ESVO/esvo_core/src/esvo_Mapping.cpp:L64-L77`

## 事件相机 vs 帧相机的时间特性

| 维度 | 帧相机 (30fps) | 事件相机 (DVS/DAVIS) |
|------|---------------|---------------------|
| 采样方式 | 全局快门 / 卷帘快门，固定帧率 | 异步逐像素，μs 级时间戳 |
| 时间分辨率 | 33 ms（帧间盲区） | ~1 μs（连续） |
| 数据率 | 恒定（分辨率 × fps × 8bit） | 稀疏（仅变化像素，典型 10-100k evts/s） |
| 动态范围 | ~60 dB | >120 dB |
| 运动模糊 | 严重（高速场景） | 几乎无（时间分辨率极高） |
| 弱纹理鲁棒性 | 差（依赖特征纹理） | 好（边缘触发，天然提取几何结构） |
| 前端数据类型 | 灰度图像矩阵 | Time Surface 2D 矩阵（从事件流生成） |
| 立体匹配方式 | 帧间特征匹配 / 灰度块匹配 | TS 块匹配（ZNCC，利用事件边缘信息） |

## Agent 实现提示

### 适用场景

- 高速运动场景（>30fps 帧相机无法胜任的快速相机运动）
- 极端光照条件（从暗室到户外强光均可，DVS >120dB 动态范围）
- 无人机 / 机器人高速避障（事件相机低延迟 + 高时间分辨率）
- 不适用：缺乏纹理边缘的空旷场景（事件相机依赖边缘触发）

### 输入输出契约

- **输入**：双目事件流（`dvs_msgs::EventArray`，左右独立）+ 双目 Time Surface（`sensor_msgs::Image`，由 esvo_time_surface 节点生成）+ 相机标定（CameraSystem，含内参、外参、基线）
- **输出**：稀疏逆深度图（`DepthMap`，含 invDepth/variance/cost/age）、3D 点云（`PointCloudXYZ`）、6-DoF 位姿
- **坐标约定**：
  - 图像坐标：校正后的像素坐标（`x_rect`），通过 `getRectifiedUndistortedCoordinate` 获取
  - 匹配在**原始 TS 图像**的整数像素坐标执行（floor 取整）
  - 逆深度 $\rho = 1/z$，在左相机坐标系下定义
  - 位姿：`Transformation`（使用 kindr 库的四元数+平移）

### 实现骨架（伪代码）

```pseudo
class ESVO_Frontend:
    def __init__(self, cam_system):
        self.event_bm = EventBM(cam_system, patch_x=25, patch_y=25,
                                 min_disp=3, max_disp=40, step=1,
                                 zncc_thresh=0.1)
        self.depth_solver = DepthProblemSolver(cam_system, dp_config,
                                                jacobian_method=NUMERICAL)
        self.depth_fusion = DepthFusion(cam_system, dp_config)
        self.ts_history = TimeSurfaceHistory()

    def mapping(self, events_left, events_right, ts_left, ts_right, ts_time):
        # 1. 事件切片：取时间窗口内的左右事件
        half_slice = 0.001  # 1ms
        events_slice = select_events_in_window(events_left, ts_time, half_slice)

        # 2. 对每个事件做双目块匹配
        match_pairs = []
        for event in events_slice:
            x_rect = rectify(event.x, event.y)  # 畸变校正
            if out_of_image(x_rect): continue
            patch_left = extract_patch(ts_left, x_rect, 25, 25)
            if is_degenerate(patch_left): continue  # >95% 像素 TS < 1

            # 极线搜索
            for disp in range(min_disp, max_disp, step):
                x_right = (x_rect[0] - disp, x_rect[1])
                patch_right = extract_patch(ts_right, x_right, 25, 25)
                cost = 0.5 * (1 - NCC_normalized(patch_left, patch_right))
                if cost < best_cost:
                    best_cost = cost; best_disp = disp

            if best_cost < ZNCC_threshold:
                depth = baseline * fx / best_disp
                match_pairs.append(EventMatchPair(x_left, x_right, t, invDepth, cost))

        # 3. 非线性深度优化（单逆深度 LM）
        for pair in match_pairs:
            depth_solver.optimize(pair)  # 1-DoF 逆深度优化

        # 4. 深度融合 + 正则化
        depth_map = depth_fusion.fuse(match_pairs)

        # 5. 发布深度图给 tracking
        publish_depth_map(depth_map, ts_time)

    def tracking(self, depth_map_ref, ts_cur, ts_time):
        # 直接法位姿追踪：最小化 TS 光度残差
        # 详见 esvo_Tracking::TrackingLoop()
        T_world_cur = optimize_pose_LM(depth_map_ref, ts_cur)
        publish_pose(T_world_cur, ts_time)
```

### 关键源码片段

**Event Block Matching 核心** (`raw/codes/ESVO/esvo_core/src/core/EventBM.cpp:L80-L168`)：
```cpp
bool EventBM::match_an_event(
    dvs_msgs::Event* pEvent,
    std::pair<size_t, size_t>& pDisparityBound,
    EventMatchPair& emPair) {

    size_t lowDisparity = pDisparityBound.first;
    size_t upDisparity  = pDisparityBound.second;

    // 畸变校正并 floor 坐标
    Eigen::Vector2d x_rect = camSysPtr_->cam_left_ptr_->
        getRectifiedUndistortedCoordinate(pEvent->x, pEvent->y);

    // 边界检查
    if(x_rect(0) < 0 || x_rect(0) > camSysPtr_->cam_left_ptr_->width_ - 1) return false;

    Eigen::Vector2i x1(std::floor(x_rect(0)), std::floor(x_rect(1)));
    if(!isValidPatch(x1, x1_left_top)) return false;

    // 提取左 TS patch
    Eigen::MatrixXd patch_src = pStampedTsObs_->second.TS_left_.block(
        x1_left_top(1), x1_left_top(0), patch_size_Y_, patch_size_X_);

    // 噪声检查：>95% 像素 TS 值 < 1 则为退化区
    if((patch_src.array() < 1).count() > 0.95 * patch_src.size())
        return false;

    // 粗搜索 + 精搜索（两级视差搜索）
    if(!epipolarSearching(min_cost, bestMatch, bestDisp, patch_dst,
            lowDisparity, upDisparity, step_, x1, patch_src, bUpDownConfiguration_))
        return false;

    // ZNCC 阈值判定
    if(min_cost <= ZNCC_Threshold_) {
        emPair.invDepth_ = 1.0 / (baseline_ * fx / disparity);
        emPair.cost_ = min_cost;
        return true;
    }
    return false;
}
```

### 实现注意事项

- **Patch 边界约束**：patch 不能触及图像边界行列，因为非线性优化中的双线性插值需要访问邻像素（`isValidPatch` 要求 `left_top ≥ 1` 且 `right_bottom ≤ width-1`）
- **Coarse + Fine 两级搜索**：粗搜索 step>1 时找到的局部最小要求**左右邻域都有有效匹配**才认为成功，防止搜索到 TS 边界外产生的虚假极小
- **Up-Down 配置**：支持水平/垂直双目配置（`bUpDownConfiguration_`），影响视差方向的搜索方式
- **事件密度 vs TS 更新频率**：TS 衰减常数 τ 和 TS 更新频率需要匹配事件速率。τ 太小 → TS 图像太稀疏；τ 太大 → TS 包含过时事件产生重影
- **深度噪声**：单个事件匹配产生的深度含有较大噪声（事件离散性 + TS 分辨率），必须经过非线性优化 + 多帧融合才能使用
- **数值稳定性**：ZNCC 归一化中除去了均值方差，对 TS 的绝对亮度不敏感，使匹配对衰减常数 τ 的选取具有鲁棒性
