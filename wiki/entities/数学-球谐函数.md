# 球谐函数

> 在 3D Gaussian Splatting 中用低阶基函数编码视角相关颜色，使同一个 3D Gaussian 可以从不同观察方向渲染出不同 RGB。

## 定义

球谐函数（Spherical Harmonics, SH）是一组定义在单位球面上的正交基函数，可把依赖观察方向的颜色函数展开为有限阶系数。在 3DGS/MonoGS 中，每个 Gaussian 不只保存一个固定 RGB，而是保存一组 SH 系数；渲染时根据相机到 Gaussian 的观察方向 `v` 计算颜色。

## 颜色模型

对某个 Gaussian，视角相关颜色可写成：

```text
c(v) = sum_{l=0..L} sum_{m=-l..l} c_lm Y_lm(v)
```

其中：

- `v` 是从 Gaussian 指向相机的单位方向。
- `Y_lm(v)` 是第 `l` 阶、第 `m` 个球谐基函数。
- `c_lm` 是 RGB 三通道的 SH 系数。
- `L` 是最高阶数，即 `sh_degree`。

阶数为 `L` 时，每个颜色通道需要 `(L + 1)^2` 个系数。0 阶只有 DC 分量，不表达视角变化；3 阶有 16 个系数，能表达更复杂的高光和视角相关外观。

## MonoGS 用法

MonoGS 渲染时会把 `pc.get_features` 重排为 `[N, 3, (max_sh_degree+1)^2]`，再调用：

```text
sh2rgb = eval_sh(active_sh_degree, sh_coeffs, view_direction)
color = clamp_min(sh2rgb + 0.5, 0)
```

`use_spherical_harmonics` 打开时使用较高阶 SH；关闭时退化为 `sh_degree=0`，即每个 Gaussian 只有与视角无关的基础颜色。

## 工程取舍

- 高阶 SH 提升视角相关外观表达，适合有反光、非朗伯表面或视角变化明显的场景。
- 高阶 SH 增加每个 Gaussian 的参数量和优化难度，SLAM 中可能放大位姿误差带来的颜色漂移。
- 单目或实时性优先场景可以先用 0 阶颜色，保证几何和位姿稳定后再提高外观表达能力。
